01 — Фундамент

Як браузер читає CSS

Перш ніж писати CSS — зрозумій що відбувається під капотом. Браузер проходить 6 кроків від завантаження HTML до пікселів на екрані. Знаючи це, розумієш чому деякі операції повільні.

Що відбувається на кожному кроці
1
DOM + CSSOM → Render Tree
Браузер будує DOM з HTML і CSSOM з CSS. Потім об'єднує в Render Tree — тільки видимі елементи з їх стилями. display: none — елемент не входить в Render Tree взагалі.
2
Layout (Reflow) — найдорожчий
Браузер обраховує розміри і позиції всіх елементів. Зміна width, height, margin, padding, font-size — тригерить повний reflow. Дорого! Може вплинути на весь документ.
3
Paint (Repaint)
Браузер малює пікселі: кольори, тіні, border. Зміна color, background, box-shadow тригерить repaint. Дешевше за reflow, але все одно немало.
4
Composite — GPU, найдешевший ✅
Тільки transform і opacity обробляються прямо на GPU без reflow/repaint. Саме тому анімуй тільки їх — отримаєш стабільні 60fps.
Дорогі операції: анімація width/height/top/left/margin/padding → reflow кожен кадр → лагає. Заміна: transform: translateX/Y/scale — без reflow.

05 — Шари

Z-index і Stacking Context

Z-index — хто поверх кого. Але є пастка: stacking context. Новий контекст означає що z-index ізолюється всередині нього — і більший z-index ззовні не допоможе.

Stacking order — порядок без z-index
/* Без z-index браузер малює в такому порядку (знизу вгору): */

/* 1. background і border кореневого елемента */
/* 2. Блоки в нормальному потоці (block) */
/* 3. Float елементи */
/* 4. Inline елементи в нормальному потоці */
/* 5. Позиційовані елементи (position != static) */

/* Z-index працює тільки на позиційованих елементах! */
.overlay {
  position: relative; /* або absolute, fixed, sticky */
  z-index: 10;         /* без position — не працює */
}
z-index: 1.box-a
z-index: 2.box-b
z-index: 3.box-c (зверху)
Що створює новий stacking context
/* Новий stacking context виникає якщо елемент має: */
.element {
  position: absolute /* + z-index != auto */
  opacity: 0.99        /* будь-яке значення крім 1 */
  transform: scale(1)   /* будь-який transform */
  filter: blur(0px)     /* будь-який filter */
  will-change: transform /* підказка браузеру */
  isolation: isolate    /* явне створення контексту */
}

/* ПРОБЛЕМА: */
.card   { position: relative; z-index: 1; }
.modal  { position: fixed;    z-index: 9999; }
/* Якщо .card має transform: translateZ(0) → він створює
   stacking context, і .modal з z-index 9999 може
   опинитися ПІД .card (якщо .card пізніше в DOM) */

/* РІШЕННЯ: isolation: isolate на батькові */
.app-root { isolation: isolate; }
Практичне правило z-index: використовуй шкалу 10-20-30 або 100-200-300 — між значеннями залишається місце для вставки. Ніколи не пиши z-index: 9999999 — це ознака незнання як працює stacking context.
06 — Переповнення

Overflow

Overflow визначає що відбувається коли вміст більший за контейнер. Правильне overflow рятує від витікаючих елементів і горизонтального скролу.

Значення overflow
overflow: visible (default)
Контент виходить за межі і залишається видимим
overflow: hidden
Контент виходить за межі і залишається видимим
overflow: scroll
Контент виходить за межі і залишається видимим
overflow: auto
Контент виходить за межі і залишається видимим. Тут більше тексту щоб показати auto скролл
/* Окремі осі */
overflow-x: hidden;  /* горизонталь */
overflow-y: auto;    /* вертикаль */

/* Типові кейси */
/* Фіксована висота з прокруткою */
.sidebar { height: 100vh; overflow-y: auto; }

/* Кнопки і бейджі не переносять текст */
.badge { white-space: nowrap; overflow: hidden; }

/* Горизонтальний скрол для таблиці */
.table-wrap { overflow-x: auto; }

/* Запобігти горизонтальному скролу на мобільному */
body { overflow-x: hidden; }

/* overflow: hidden ТАКОЖ обрізає box-shadow!
   Якщо тінь зникає — перевір overflow батька */
07 — Фони

Background — повний арсенал

Background — не просто колір. Це зображення, градієнти, позиція, розмір, повтор. Можна накладати кілька шарів через кому.

Всі властивості background
.element {
  /* Колір — завжди під зображенням */
  background-color: hsl(220 90% 50%);

  /* Зображення */
  background-image: url('hero.jpg');

  /* Позиція: ключові слова або % або px */
  background-position: center;        /* = 50% 50% */
  background-position: top right;     /* кут */
  background-position: 20px 40px;     /* зміщення */

  /* Розмір */
  background-size: cover;    /* заповнює, обрізає */
  background-size: contain;  /* вписується, може бути пробіл */
  background-size: 100% auto; /* точний контроль */

  /* Повтор */
  background-repeat: no-repeat; /* без повтору */
  background-repeat: repeat-x;  /* тільки горизонтально */

  /* Прив'язка до скролу */
  background-attachment: fixed;  /* parallax ефект */
  background-attachment: scroll; /* default */

  /* Область відображення */
  background-clip: text;        /* gradient text! */
  background-clip: border-box;  /* default */
  background-clip: padding-box; /* без border */
}

/* ─── ШОРТКАТ background ─── */
.hero {
  background: url('hero.jpg') center/cover no-repeat #1a1a2e;
  /*          image           position/size  repeat   color */
}

/* ─── КІЛЬКА ШАРІВ (порядок: перший — зверху) ─── */
.card {
  background:
    linear-gradient(rgba(0,0,0,0.4), transparent),  /* overlay */
    url('photo.jpg') center/cover;                   /* photo */
}

/* ─── GRADIENT TEXT ─── */
.gradient-text {
  background: linear-gradient(135deg, #667eea, #764ba2);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  background-clip: text;
}

08 — Трансформації

CSS Transforms

Transform змінює вигляд елемента без впливу на потік — сусіди не зрушуються. Відбувається на GPU — ідеально для анімацій. Наведи на картки.

Основні функції — наведи щоб побачити
translate
translate(12px,-8px)
scale
scale(1.5)
rotate
rotate(135deg)
skew
skewX(20deg)
scaleY
scaleY(0.5)
3D rotate
rotateY(60deg)
/* Кілька трансформацій — через пробіл */
.card:hover {
  transform: translateY(-4px) scale(1.02);
}

/* transform-origin — точка відліку трансформації */
transform-origin: center;       /* default */
transform-origin: top left;     /* розкривається з кута */
transform-origin: 50% 0%;       /* зверху по центру */

/* Сучасний синтаксис: окремі властивості */
translate: 0 -4px;     /* замість transform: translateY(-4px) */
scale: 1.02;            /* замість transform: scale(1.02) */
rotate: 45deg;          /* замість transform: rotate(45deg) */
/* Перевага: можна анімувати незалежно! */

/* will-change — підказка браузеру заздалегідь */
.animated {
  will-change: transform; /* пром'якшує перший кадр */
}
/* Але не зловживай — займає пам'ять GPU */

11 — Дебаг

Як дебажити CSS

CSS-баг не кидає помилку в консоль. Треба вміти знаходити проблеми самому. Ось арсенал.

Методи діагностики
01
Outline все
* {
  outline: 1px solid
  rgba(255,0,0,0.3);
}
Підсвічує всі елементи. Бачиш реальні розміри і перетини без впливу на layout (outline не займає місця).
02
DevTools → Inspect
F12 → клік на елемент. Box model зправа, computed styles, список правил з файлами. Перекреслені стилі — перекриті іншими.
03
Background colors
.debug {
  background: rgba(255,0,0,.1);
  border: 1px solid red;
}
Швидко зрозуміти де закінчується елемент і чому він не там де треба.
04
DevTools → Layers
Показує stacking contexts і GPU шари. Корисно коли z-index поводиться дивно або анімація лагає.
Типові баги і виправлення
/* ❌ Горизонтальний скрол на мобільному */
body { overflow-x: hidden; }
/* Причина: елемент з width > 100vw або від'ємний margin */

/* ❌ Зображення не вписується */
img { max-width: 100%; height: auto; display: block; }

/* ❌ Елемент не центрується */
/* Причина: margin: auto НЕ ПРАЦЮЄ на inline елементах */
.el { display: block; margin: 0 auto; width: fit-content; }

/* ❌ z-index не працює */
/* Причина: без position не діє */
.el { position: relative; z-index: 10; }

/* ❌ box-shadow обрізається */
/* Причина: overflow: hidden на батькові */
.parent { overflow: visible; } /* або padding замість overflow */

/* ❌ :hover не спрацьовує на touch-пристроях */
/* Рішення: завжди є базовий стиль, hover — покращення */

/* ❌ Стиль не застосовується */
/* Перевір: специфічність, !important, опечатку в class */
/* DevTools → Computed → знайди властивість */

/* ❌ Flex items не переносяться */
.container { flex-wrap: wrap; } /* потрібен flex-wrap */

/* ❌ Висота 100% не працює */
/* Причина: батько не має заданої висоти */
html, body { height: 100%; }
.child { height: 100%; } /* тепер спрацює */
