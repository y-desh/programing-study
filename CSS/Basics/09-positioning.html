Позиціонування relative · absolute

Чому НЕ треба використовувати absolute для layout. Єдиний правильний кейс — overlays. position: relative на батькові, absolute на дитині. ::before pseudo-element.

01 — Навіщо positioning

Absolute і Relative — тільки для overlays

EdRow категоричний: "НЕ використовуй absolute і relative для layout. Це спокуслива пастка для новачків." Позиціонування — тільки коли треба накласти елемент на інший.

Чому absolute небезпечно для layout
/* ❌ ПОГАНО — layout через absolute */
#test  { position: absolute; right: 5px; }
#test2 { position: absolute; right: 10px; }

/* Проблеми:
   1. Не адаптується до розміру екрана
   2. Overlap з іншими елементами
   3. Якщо видалити один — інші не зсуваються
   4. Важко підтримувати */

/* ✅ Правильний кейс — overlay */
/* Текст поверх зображення 
   Бейдж в куті картки
   Modal поверх контенту */
EdRow: "Flex виправляє себе автоматично — видали один елемент, інші автоматично перебудуються. Absolute — ні. Тому Flex завжди кращий для layout."
02 — Правильний кейс

relative + absolute для overlays

Текст поверх зображення
/* Батько — relative (точка відліку) */
.hot-card {
  position: relative;
  border-radius: 8px;
  overflow: hidden;
}

/* Дитина — absolute (відносно батька) */
.hot-overlay {
  position: absolute;
  bottom: 0;    /* притиснути до низу */
  left: 0;
  right: 0;    /* на всю ширину */

  padding: 1rem;
  background: rgba(0,0,0,0.7);
  font-weight: bold;
  color: white;
}
Overlay текст поверх зображення
::before псевдоелемент для hover ефекту
/* Альтернатива — без HTML, через CSS */
.masonry-item {
  position: relative;
}

.masonry-item::before {
  content: "";       /* обов'язково! */
  position: absolute;
  top: 0; left: 0;
  right: 0; bottom: 0;
  background: rgba(0,0,0,0.5);
  opacity: 0;
  transition: opacity 0.3s ease;
}

/* Hover — показуємо overlay */
.masonry-item:hover::before {
  opacity: 1;
}

🎯 Практика
1. Зроби картку з зображенням і overlay текстом внизу (position: relative + absolute, bottom:0)
2. Додай hover ефект через ::before — темний overlay з'являється при наведенні
3. Зроби бейдж "NEW" в правому верхньому куті картки через absolute top:10px right:10px
