01 — Основа основ

Box Model

Кожен HTML-елемент — це прямокутна коробка. Завжди. Браузер малює кожен елемент як чотири концентричних шари. Якщо ти не розумієш Box Model — ти будеш гадати чому відступи «не працюють».

Візуальна структура елемента

margin
border
padding
content
content — сам вміст
padding — простір всередині
border — рамка елемента
margin — простір зовні

❌ content-box (default)

.box { width: 200px; padding: 20px; border: 2px solid; } /* РЕАЛЬНА ширина: 200 + 20 + 20 + 2 + 2 = 244px 😱 */

✅ border-box (правильно)

*, *::before, *::after { box-sizing: border-box; } .box { width: 200px; padding: 20px; /* рівно 200px ✅ */ }
Правило #1: Перший рядок кожного CSS файлу — завжди box-sizing: border-box на всіх елементах. Без виключень.
02 — Поведінка в потоці

Display

Властивість display визначає як елемент розміщується в потоці документа і як поводяться його діти. Це найбільш впливова властивість в CSS після кольору.

Порівняння режимів відображення

block
div №1
div №2 — займає всю ширину
div №3 — кожен з нового рядка
inline
Це текст з <span> і ще один <a> — всі в одному рядку, width/height не діє
inline-block
box 1 box 2 box 3 ← є width/height, але в рядку

Таблиця значень

display: block; /* div, p, h1..h6, section — вся ширина, новий рядок */ display: inline; /* span, a, strong — в рядку, width/height ігнор. */ display: inline-block; /* гібрид: в рядку + є розміри */ display: none; /* прихований + не займає місця */ display: flex; /* flex-контейнер для дітей */ display: grid; /* grid-контейнер для дітей */
Чому span не реагує на width? Тому що він inline за замовчуванням. Додай display: inline-block або display: block — і width запрацює.

04 — Одновимірне розміщення

Flexbox

Flex — це система для розміщення елементів вздовж однієї осі (рядок або стовпець). Вмикається на батькові, діє на дітей. Вирішує 80% задач верстки.

Осі Flexbox

.container { display: flex; flex-direction: row; /* → головна вісь горизонтальна */ flex-direction: column; /* ↓ головна вісь вертикальна */ } /* justify-content — вирівнювання по ГОЛОВНІЙ осі */ /* align-items — вирівнювання по ПОПЕРЕЧНІЙ осі */

Ментальна модель: уяви flex-контейнер як резинку. justify-content розтягує резинку горизонтально між дітьми. align-items вирівнює їх по вертикалі. flex: 1 на дитині — вона «з'їдає» весь залишок.

Flex vs Grid — коли що обирати

Flex — коли:
→ елементи в один рядок
→ навбар з лого і кнопками
→ центрувати що-небудь
→ карточки в ряд з wrap
→ розмір дітей динамічний
Grid — коли:
→ розкладка всієї сторінки
→ потрібні і рядки і колонки
→ елементи мають span
→ dashboard layout
→ точне позиціювання
06 — Хто виграє

Каскад і Специфічність

Коли два правила CSS конфліктують — браузер вираховує «вагу» кожного селектора. Хто важчий — той виграє. Це і є специфічність.

Практичний приклад

h1 { color: black; } /* вага: 1 */ .title { color: blue; } /* вага: 10 */ .nav .title { color: green; } /* вага: 20 */ #page-title { color: purple; } /* вага: 100 — виграє */ /* Якщо однакова вага — виграє той що пізніше у файлі */ p { color: red; } p { color: blue; } /* ← цей виграє */
!important — зброя масового ураження. Кожен раз як пишеш !important — ти розриваєш каскад. Наступного разу щоб перекрити — треба ще один !important. Так проєкт стає некерованим. Вирішуй проблему через правильну специфічність.
07 — Міри і розміри

Одиниці виміру

Вибір правильної одиниці — різниця між адаптивним і «зламаним» дизайном.

px
Абсолютні пікселі. Не масштабуються.
border, shadow, icon size
rem
Відносно шрифту <html>. 1rem = 16px за замовч.
font-size, padding, margin
em
Відносно шрифту батька. Каскадується!
padding пов'язаний з текстом
%
Відсоток від батьківського елемента.
width, height контейнерів
vw/vh
Відсоток від ширини/висоти вікна браузера.
hero, fullscreen секції
fr
Fraction — частка вільного простору в Grid.
тільки в grid-template-columns

clamp() — найрозумніша одиниця

/* clamp(мінімум, ідеал, максимум) */ h1 { font-size: clamp(1.5rem, 5vw, 3rem); /* на телефоні: 1.5rem */ /* на планшеті: десь між */ /* на моніторі: 3rem */ /* без жодного медіазапиту! */ } p { max-width: 65ch; /* 65 символів = ідеальна ширина рядка */ }
Правило ментора: font-size → rem. Spacing → rem або px. Width контейнерів → % або fr. Border, shadow → px. Адаптивний текст → clamp().
