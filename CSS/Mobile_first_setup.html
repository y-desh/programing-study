01 — Підхід

Навіщо Mobile First?

Mobile First — це не просто спосіб писати медіазапити. Це мислення: спочатку найпростіша версія для найменшого екрана, потім поступове розширення. Браузер завантажує мінімум CSS і далі додає шари.

Реальні цифри
60%
світового трафіку — мобільні пристрої
3G
частина юзерів досі на повільному інтернеті
53%
залишають сайт якщо він вантажиться більше 3с
Mobile First vs Desktop First — різниця в голові
✅ Mobile First
/* Базові стилі = мобільний */
.grid {
  display: flex;
  flex-direction: column;
  gap: 16px;
}
/* Потім розширюємо */
@media (min-width: 768px) {
  .grid {
    flex-direction: row;
  }
}
min-width → додаємо складність
VS
❌ Desktop First
/* Базові стилі = десктоп */
.grid {
  display: flex;
  flex-direction: row;
  gap: 32px;
}
/* Потім "ламаємо" для малих */
@media (max-width: 767px) {
  .grid {
    flex-direction: column;
  }
}
max-width → боремося зі складністю
Чому min-width кращий: браузер завантажує базові стилі завжди. При mobile-first базові стилі прості — мобільний контекст. Desktop-версія отримує розширення поверх. При desktop-first мобільний повинен скасовувати складні десктопні стилі — це дорожче і брудніше.

02 — Крок 1

Фундамент: Reset + Base

Перший файл в будь-якому проєкті — це нульова точка. Скидаємо браузерні дефолти, вирівнюємо поведінку між браузерами, встановлюємо глобальні правила. Без цього кожен елемент — сюрприз.

1
Глобальний box-sizing
Першочергово — і ніколи не забувай
/* Рядок №1 у кожному CSS проєкті */
*, *::before, *::after {
  box-sizing: border-box;
}
2
Modern CSS Reset
Мінімальний reset — не normalize.css (1500 рядків), а лише що реально потрібно
/* Прибрати відступи браузера */
body, h1, h2, h3, h4, p,
figure, blockquote, dl, dd {
  margin: 0;
}

/* Списки без стилів — якщо role="list" */
ul[role="list"], ol[role="list"] {
  list-style: none;
  padding: 0;
}

/* Плавний скрол для всього */
html {
  scroll-behavior: smooth;
  /* Але вимикаємо якщо юзер проти */
}

/* Зображення не виходять за батька */
img, picture, video, canvas, svg {
  display: block;
  max-width: 100%;
}

/* Форми успадковують шрифт */
input, button, textarea, select {
  font: inherit;
}

/* Текстові елементи не виходять за межі */
p, h1, h2, h3, h4, h5, h6 {
  overflow-wrap: break-word;
}

/* Прибираємо анімації якщо юзер так налаштував ОС */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
3
Base body стилі
Глобальні дефолти — колір, шрифт, background
body {
  /* Шрифт для всього документа */
  font-family: var(--font-body);
  font-size: var(--text-base);
  line-height: var(--leading-normal);

  /* Кольори */
  background-color: var(--color-bg);
  color: var(--color-text);

  /* Антиаліасинг — шрифти виглядають краще */
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;

  /* Мінімальна висота = вся висота екрана */
  min-height: 100vh;
}
Порядок підключення файлів важливий: reset.css → tokens.css → base.css → layout.css → components.css → utilities.css. Кожен наступний файл може перевизначати попередній.
03 — Крок 2

Design Tokens: CSS Variables

Design tokens — це іменовані значення, які зберігають всі рішення дизайн-системи. Колір, розмір, відступ, радіус — все в одному місці. Зміни поширюються на весь проєкт автоматично.

Повний токен-файл проєкту
:root {

  /* ─── КОЛЬОРИ ─────────────────────────── */
  /* Палітра (не використовуй напряму) */
  --blue-50:   hsl(214 100% 97%);
  --blue-500:  hsl(214 90%  55%);
  --blue-900:  hsl(214 90%  15%);

  /* Семантичні токени (використовуй їх) */
  --color-bg:         #ffffff;
  --color-bg-subtle:  #f8fafc;
  --color-text:       #0f172a;
  --color-text-muted: #64748b;
  --color-border:     #e2e8f0;
  --color-primary:    var(--blue-500);
  --color-primary-bg: var(--blue-50);
  --color-danger:     hsl(0 84% 60%);
  --color-success:    hsl(142 71% 45%);

  /* ─── ТИПОГРАФІКА ─────────────────────── */
  --font-body:    'Inter', system-ui, sans-serif;
  --font-heading: 'Cal Sans', 'Inter', sans-serif;
  --font-mono:    'JetBrains Mono', monospace;

  /* Шкала розмірів — fluid */
  --text-xs:   clamp(0.7rem,  1vw,   0.75rem);
  --text-sm:   clamp(0.8rem,  1.5vw, 0.875rem);
  --text-base: clamp(1rem,    2vw,   1rem);
  --text-lg:   clamp(1.1rem,  2.5vw, 1.25rem);
  --text-xl:   clamp(1.25rem, 3vw,   1.5rem);
  --text-2xl:  clamp(1.5rem,  4vw,   2rem);
  --text-3xl:  clamp(2rem,    5vw,   3rem);
  --text-4xl:  clamp(2.5rem,  7vw,   4.5rem);

  --leading-normal: 1.6;
  --leading-loose:  1.8;

  /* ─── ВІДСТУПИ ────────────────────────── */
  /* Базова одиниця = 4px */
  --space-1:  4px;
  --space-2:  8px;
  --space-3:  12px;
  --space-4:  16px;
  --space-6:  24px;
  --space-8:  32px;

  --space-24: 96px;

  --radius-lg: 16px;
  --radius-xl: 24px;

  --shadow-lg: 0 16px 48px rgba(0,0,0,0.12);

  --duration-normal: 250ms;
  --duration-slow:   400ms;

  /* ─── BREAKPOINTS (для JS, в CSS використовуй напряму) */
  --bp-sm:  640px;
  --bp-md:  768px;
  --bp-lg:  1024px;
  --bp-xl:  1280px;
  --bp-2xl: 1536px;

  /* ─── ТЕМНА ТЕМА ─────────────────────── */
}

[data-theme="dark"] {
  --color-bg:         #0f1117;
  --color-bg-subtle:  #161b27;
  --color-text:       #e2e8f0;
  --color-text-muted: #64748b;
  --color-border:     #1e293b;
}
Два рівні токенів: спочатку «примітивні» (--blue-500 = #3b82f6), потім «семантичні» (--color-primary = var(--blue-500)). Компоненти посилаються тільки на семантичні. Тоді для dark theme достатньо перевизначити 5-6 семантичних токенів.
04 — Крок 3

Типографічна система

Після токенів — базові стилі для тексту. Всі заголовки, абзаци, посилання. Mobile first: на малому екрані текст менший, на великому — більший. Автоматично через clamp().

base/typography.css
/* ─── ЗАГОЛОВКИ ─── */
h1, h2, h3, h4, h5, h6 {
  font-family: var(--font-heading);
  line-height: var(--leading-tight);
  color: var(--color-text);
  /* margin-block — сучасний замість margin-top/bottom */
  margin-block: var(--space-6) var(--space-4);
}

h1 { font-size: var(--text-4xl); font-weight: 800; }
h2 { font-size: var(--text-3xl); font-weight: 700; }
h3 { font-size: var(--text-2xl); font-weight: 700; }
h4 { font-size: var(--text-xl);  font-weight: 600; }

/* ─── ПАРАГРАФИ ─── */
p {
  font-size: var(--text-base);
  line-height: var(--leading-loose);
  color: var(--color-text);
  max-width: 65ch;  /* оптимальна ширина рядка */
}

/* ─── ПОСИЛАННЯ ─── */
a {
  color: var(--color-primary);
  text-decoration: underline;
  text-underline-offset: 3px; /* відстань підкреслення від тексту */
  transition: color var(--duration-fast) var(--ease);
}
a:hover { color: hsl(from var(--color-primary) h s calc(l - 15%)); }

/* ─── ДОПОМІЖНІ ─── */
small    { font-size: var(--text-sm); }
strong   { font-weight: 700; }
em       { font-style: italic; }
code     { font-family: var(--font-mono); font-size: 0.875em; }
pre code { font-size: inherit; }
clamp() — твій найкращий друг: шкала --text-xs до --text-4xl вже містить fluid значення через clamp(). Пишеш один раз в токенах — і h1 автоматично 40px на телефоні і 72px на моніторі. Жодного media query для типографіки.
05 — Крок 4

Layout System

Layout — це скелет. Container обмежує ширину, wrapper центрує, grid розставляє елементи. Пишеться один раз і переноситься між проєктами.

layout/container.css — контейнер
/* Основний контейнер */
.container {
  /* min() — не більше ніж max-width, але і не більше 100% - padding */
  width: min(100% - 32px, 1200px);
  margin-inline: auto;   /* сучасний замість margin: 0 auto */
}

/* Варіанти ширини */
.container-sm  { width: min(100% - 32px, 640px); margin-inline: auto; }
.container-md  { width: min(100% - 32px, 768px); margin-inline: auto; }
.container-xl  { width: min(100% - 32px, 1440px);margin-inline: auto; }
.container-full{ width: 100%; padding-inline: var(--space-4); }

/* На мобільному padding менший */
@media (min-width: 768px) {
  .container {
    width: min(100% - 64px, 1200px);
  }
}
layout/grid.css — сітка

06 — Крок 5

Компоненти

Компонент — це самодостатній CSS-модуль. Він стилізує конкретний UI-елемент і не залежить від оточення. Пишеться mobile first: дефолт для телефону, розширення для більших екранів.

Кнопка — еталонний компонент
/* components/button.css */

.btn {
  /* Скидаємо браузерні стилі */
  appearance: none;
  border: none;
  cursor: pointer;
  text-decoration: none;

  /* Базова структура */
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--space-2);

  /* Mobile розмір (менший) */
  padding: var(--space-3) var(--space-4);
  font-size: var(--text-sm);
  min-height: 44px;   /* мінімум для touch target */
  min-width: 44px;

  /* Стиль */
  font-family: var(--font-body);
  font-weight: 600;
  border-radius: var(--radius);
  white-space: nowrap;

  /* Анімація */
  transition:
    background var(--duration-fast) var(--ease),
    transform var(--duration-fast) var(--ease),
    box-shadow var(--duration-fast) var(--ease);
}

/* Активний стан — натискання */
.btn:active { transform: scale(0.97); }

/* Заблокована кнопка */
.btn:disabled {
  opacity: 0.5;
  cursor: not-allowed;
  pointer-events: none;
}

/* ─── ВАРІАНТИ ─── */
.btn-primary {
  background: var(--color-primary);
  color: white;
}
.btn-primary:hover {
  background: hsl(from var(--color-primary) h s calc(l - 8%));
  box-shadow: var(--shadow);
}

.btn-secondary {
  background: transparent;
  color: var(--color-text);
  border: 1px solid var(--color-border);
}
.btn-secondary:hover {
  background: var(--color-bg-subtle);
}

.btn-ghost {
  background: transparent;
  color: var(--color-primary);
}
.btn-ghost:hover {
  background: var(--color-primary-bg);
}

/* ─── РОЗМІРИ ─── */
.btn-sm { padding: var(--space-2) var(--space-3); font-size: var(--text-xs); }
.btn-lg { padding: var(--space-4) var(--space-8); font-size: var(--text-lg); }
.btn-full { width: 100%; }  /* на мобільному часто потрібно */

/* На desktop кнопки трохи більші */
@media (min-width: 768px) {
  .btn {
    padding: var(--space-3) var(--space-6);
    font-size: var(--text-base);
  }
}

08 — Структура

Архітектура CSS файлів

CSS без структури — хаос після 500 рядків. Добра архітектура означає: знаєш де шукати і знаєш куди додавати. Ось підхід на основі 7-1 pattern спрощений до практики.

Структура папок
📁 styles/ 📁 base/ _reset.css ← скидання _tokens.css ← змінні _typography.css _base.css ← body, html 📁 layout/ _container.css _grid.css _header.css _footer.css 📁 components/ _button.css _card.css _form.css _modal.css _navbar.css 📁 pages/ _home.css _about.css 📁 utilities/ _spacing.css _flex.css _text.css main.css ← @import всього 
/* main.css — точка входу */

/* 1. База */
@import 'base/_reset.css';
@import 'base/_tokens.css';
@import 'base/_typography.css';
@import 'base/_base.css';

/* 2. Layout */
@import 'layout/_container.css';
@import 'layout/_grid.css';
@import 'layout/_header.css';
@import 'layout/_footer.css';

/* 3. Компоненти */
@import 'components/_button.css';
@import 'components/_card.css';
@import 'components/_form.css';
@import 'components/_modal.css';
@import 'components/_navbar.css';

/* 4. Сторінки */
@import 'pages/_home.css';

/* 5. Утиліти — завжди останніми */
@import 'utilities/_spacing.css';
@import 'utilities/_flex.css';
@import 'utilities/_text.css';
Шари каскаду (@layer) — сучасний підхід
/* Оголошуємо порядок шарів */
@layer reset, tokens, base, layout, components, utilities;

@layer reset {
  /* Все що в reset */
}

@layer tokens {
  :root { /* змінні */ }
}

@layer components {
  .btn { /* стилі кнопки */ }
}

@layer utilities {
  /* Утиліти завжди виграють у компонентів */
  .hidden { display: none; }
  .sr-only { /* для screen readers */ }
}

/* @layer вирішує проблему специфічності між шарами:
   utilities завжди перемагають components
   навіть якщо специфічність однакова! */
@layer — нова суперсила CSS. Каскад більше не залежить від порядку файлів і специфічності між шарами. Оголосив шар останнім — він завжди виграє. Підтримується в усіх сучасних браузерах.
09 — Підсумок

Чекліст нового проєкту

Порядок дій при старті будь-якого CSS проєкту з нуля

1
Reset + box-sizing
Перший файл. Глобальний border-box, скидання відступів, img max-width: 100%
2
Design Tokens
Кольори, шрифти, відступи, радіуси, тіні, анімації — все в :root
3
Base Typography
h1-h6, p, a, code — fluid через clamp(), line-height, max-width: 65ch
4
Layout System
Container, grid утиліти, stack/row, sticky footer — переноситься між проєктами
5
Компоненти
Кожен компонент у своєму файлі. Mobile-first: дефолт → @media min-width
6
Утиліти
.hidden, .sr-only, .truncate, .flex-center — завжди останніми
Що перевірити перед здачею
✓
Touch targets ≥ 44px
Кожна кнопка і посилання мають min-height: 44px і min-width: 44px — стандарт Apple/Google для пальців
✓
Font-size ≥ 16px на мобільному
iOS автоматично зумує якщо input має font-size менше 16px — завжди перевіряй форми
✓
Overflow-x: hidden на body
Горизонтальний скрол на мобільному — найпоширеніший баг. body { overflow-x: hidden } рятує в 90% випадків
✓
Focus visible для клавіатури
:focus-visible { outline: 2px solid var(--color-primary); outline-offset: 2px } — не прибирай outline зовсім
✓
dvh замість vh на мобільному
min-height: 100dvh враховує мобільний браузер з адресним рядком. Класичний vh — ні
✓
prefers-reduced-motion
Вимикай анімації для юзерів які налаштували це в ОС — accessibility вимога
✓
Contrast ratio ≥ 4.5:1
Текст на фоні — перевіряй через WebAIM Contrast Checker. WCAG AA мінімум для нормальних юзерів
