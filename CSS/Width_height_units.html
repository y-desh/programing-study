01 — Основа

Одиниці виміру

CSS має більше 15 одиниць виміру. Але на практиці регулярно використовуєш лише 7–8. Завдання — знати яку одиницю і де застосовувати, бо неправильний вибір ламає адаптивність.

02 — Розміри

Width і Height

Width і height — базові властивості. Але у них є особливості поведінки які постійно плутають новачків. Головне: елементи за замовчуванням адаптивні — не задавай розміри без причини.

Ключові значення width і height
width: 300px;       /* фіксована ширина */
width: 50%;         /* 50% від батька */
width: auto;        /* браузер сам (default) */
width: fit-content; /* по вмісту */
width: max-content; /* максимально без переносу */
width: min-content; /* мінімально (за найдовшим словом) */
width: 100%;        /* = ширина батька */

height: 200px;     /* фіксована висота */
height: auto;      /* за вмістом (default) ← завжди краще */
height: 100%;      /* 100% від батька — батько має бути з висотою! */
height: 100dvh;    /* 100% висоти viewport (мобільний) */
батько · 100%
width: 100%
width: 60%
60%
width: fit-content
fit-content
width: max-content
max-content — без переносу
Не задавай height без причини! Фіксована висота — причина №1 чому текст "вилазить" за блок. Завжди починай з height: auto (default) і задавай висоту тільки коли справді потрібно. Замість висоти краще використовуй padding або min-height.
Різниця auto, 100%, fit-content наочно
/* ─── WIDTH: AUTO (default для block) ──────────────── */
div {
  width: auto;
  /* Block елемент: розтягується на всю ширину батька */
  /* АЛЕ: padding і border не виходять за межі батька */
}

/* ─── WIDTH: 100% ───────────────────────────────────── */
div {
  width: 100%;
  padding: 20px;
  /* БЕЗ box-sizing: border-box → реальна ширина 100% + 40px */
  /* Тому завжди * { box-sizing: border-box } ← перший рядок */
}

/* ─── FIT-CONTENT ───────────────────────────────────── */
.tag {
  width: fit-content;
  /* Займає рівно стільки скільки вміст, але не більше */
  /* Корисно: центрувати блок по вмісту */
  margin-inline: auto; /* + це = centered по вмісту */
}

/* ─── HEIGHT 100% ───────────────────────────────────── */
.child { height: 100%; } /* НЕ ПРАЦЮЄ якщо:
.parent { } /* ← батько не має height або min-height */

/* ПРАВИЛЬНО: */
.parent { height: 400px; }    /* або min-height */
.child  { height: 100%; }     /* тепер = 400px */
03 — Межі

Min-width, Max-width, Min-height, Max-height

Min/max дозволяють задати діапазон замість фіксованого розміру. Елемент вільно масштабується всередині цього діапазону. Це основа адаптивного дизайну без media queries.

Як вони взаємодіють між собою
/* Пріоритет:  min-width > max-width > width */
.el {
  width: 50%;        /* бажана ширина */
  min-width: 200px;  /* не менше 200px (навіть якщо 50% менше) */
  max-width: 600px;  /* не більше 600px (навіть якщо 50% більше) */
}
/* Реальна ширина = clamp(200px, 50%, 600px) */
/*                        ↑ мін  ↑ ідеал ↑ макс */

/* min перемагає max! */
.broken {
  min-width: 400px;
  max-width: 200px; /* ← min виграє → буде 400px */
}
max-width: 600px — контейнер не росте далі
max-width: 600px — займає весь простір до 600px, далі не росте
min-width: 200px — не стискається нижче порогу
min-width: 200px
Найважливіші паттерни min/max
/* ── КОНТЕЙНЕР СТОРІНКИ ── */
.container {
  /* max-width обмежує зростання */
  max-width: 1200px;
  width: 100%;       /* займає все до 1200 */
  margin-inline: auto; /* центрування */
  padding-inline: 24px;/* відступи по боках */
}

/* Або сучасний варіант: */
.container {
  width: min(100% - 48px, 1200px);
  margin-inline: auto;
}

/* ── КНОПКА ── */
.btn {
  min-width: 120px;   /* мінімальна ширина */
  min-height: 44px;  /* touch target */
  padding: 10px 20px;
}

/* ── ЗОБРАЖЕННЯ ── */
img {
  max-width: 100%; /* не виходить за батька */
  height: auto;    /* зберігає пропорції */
}

/* ── TEXTAREA ── */
textarea {
  min-height: 120px; /* мінімальна висота */
  max-height: 400px; /* не виростає нескінченно */
  resize: vertical;  /* тільки вертикальний resize */
}

/* ── SIDEBAR + MAIN ── */
.sidebar {
  width: 280px;
  min-width: 240px;
  max-width: 320px;
}
.main { flex: 1; min-width: 0; } /* min-width:0 важливо! */

min-width: 0 — прихований баг у Flex
/* ПРОБЛЕМА: flex-child не стискається нижче min-content */
.flex-parent { display: flex; }
.sidebar { width: 300px; flex-shrink: 0; }
.main {
  flex: 1;
  /* Якщо є довге слово або широкий елемент —
     .main буде ширшим за доступний простір! */
}

/* РІШЕННЯ: */
.main {
  flex: 1;
  min-width: 0;  /* дозволяє стискатись нижче min-content */
  overflow: hidden; /* або overflow: auto */
}

/* Те саме стосується Grid children */
.grid-child {
  min-width: 0;  /* якщо вміст виходить за сітку */
}
min-width: 0 на flex/grid дітях — один із найпоширеніших прихованих багів. Flex і Grid елементи за замовчуванням мають min-width: auto, що не дає їм стискатися. Якщо бачиш що вміст виходить за контейнер — спробуй min-width: 0.
04 — Fluid розміри

clamp() — найрозумніша функція CSS

clamp(мін, ідеал, макс) — повертає ідеал, але не менше мін і не більше макс. Замінює кілька media queries одним рядком.

Візуалізація clamp()
clamp(1rem, 4vw, 3rem)
мін
1rem
3rem
макс
📱 малий → 1rem (мін)
↔ середній → 4vw (fluid)
🖥 великий → 3rem (макс)
/* Синтаксис */
property: clamp(мінімум, ідеальне, максимум);

/* Приклади */
font-size: clamp(1rem,    4vw,      2.5rem);
padding:   clamp(16px,   5vw,      48px);
width:     clamp(280px,  80%,      800px);
gap:       clamp(12px,   2vw,      32px);

/* Fluid типографіка без жодного @media */
h1 { font-size: clamp(1.8rem, 6vw, 5rem); }
h2 { font-size: clamp(1.4rem, 4vw, 3rem); }
p  { font-size: clamp(1rem,   2vw, 1.1rem); }

/* clamp() = min-width + max-width в одному рядку */
/* Це одне і те саме: */
.box {
  width: 50%;
  min-width: 200px;
  max-width: 600px;
}
/* = */
.box { width: clamp(200px, 50%, 600px); }
Ідеальна формула для vw: clamp(мін, (розмір_на_320px / 320) * 100vw, макс). Якщо хочеш 18px на 320px — ідеал = (18/320)*100vw = 5.625vw.
min() і max() — менш відомі сестри clamp()
/* min() — повертає МЕНШЕ з двох значень */
.container {
  width: min(100% - 48px, 1200px);
  /* = width: 100% - 48px, але не більше 1200px */
  /* = max-width: 1200px + padding 24px по боках */
  margin-inline: auto;
}

.font { font-size: min(16px, 4vw); }
/* Буде 4vw, але не більше ніж 16px */

/* max() — повертає БІЛЬШЕ з двох значень */
.padding {
  padding: max(16px, 3vw);
  /* Буде 3vw, але не менше ніж 16px */
}

.font2 { font-size: max(14px, 1.5vw); }
/* Забезпечує мінімум 14px навіть на маленьких екранах */

/* Можна вкладати! */
.el { width: min(max(200px, 40%), 800px); }
/* = не менше 200px, не більше 800px, ідеал 40% */
05 — Viewport

Viewport одиниці — vw, vh, vmin, vmax

Viewport — це видима область браузера. Viewport одиниці прив'язані до неї, а не до батьківського елемента. Корисно для fullscreen секцій і fluid типографіки.

Всі viewport одиниці
100vw = вся ширина вікна
100vh = вся висота
50vw
vmin
= менше з vw/vh
vmax
= більше з vw/vh
vw — % від ширини вікна
vh — % від висоти вікна
vmin — % від меншого (vw або vh)
vmax — % від більшого (vw або vh)
dvh/svh/lvh — динамічні (мобільний браузер)
/* Типові кейси використання */

/* Hero секція — завжди на весь екран */
.hero { min-height: 100dvh; } /* dvh для мобільного! */

/* Fluid шрифт залежний від ширини */
h1 { font-size: 8vw; } /* 8% від ширини вікна */
/* Краще з clamp: */
h1 { font-size: clamp(2rem, 8vw, 6rem); }

/* Повноекранний sidebar */
.sidebar { height: 100vh; overflow-y: auto; }

/* Modal overlay */
.overlay {
  position: fixed;
  inset: 0; /* top:0 right:0 bottom:0 left:0 */
  width: 100vw;
  height: 100vh;
}

/* vmin — квадрат що влазить в будь-яку орієнтацію */
.avatar {
  width: 20vmin;
  height: 20vmin;
}
06 — Пропорції

Aspect Ratio — фіксовані пропорції

aspect-ratio задає співвідношення сторін. Висота автоматично обраховується від ширини. Більше не треба хак з padding-bottom: 56.25%.

/* Базовий синтаксис */
.video-wrap {
  width: 100%;
  aspect-ratio: 16 / 9;  /* висота = width * 9/16 */
}

/* Зображення з фіксованими пропорціями */
.card-image {
  aspect-ratio: 4 / 3;
  overflow: hidden;
}
.card-image img {
  width: 100%;
  height: 100%;
  object-fit: cover;  /* заповнює без спотворення */
}

/* Квадратна іконка-кнопка */
.icon-btn {
  width: 44px;
  aspect-ratio: 1;  /* = 1/1 */
}

/* object-fit — як зображення заповнює контейнер */
object-fit: cover;    /* заповнює, може обрізати */
object-fit: contain;  /* вписується повністю */
object-fit: fill;     /* розтягує (спотворює) */
object-position: top center; /* де "вирізати" при cover */
07 — Стандарти

Стандарти розмірів сайту

Немає єдиних "правильних" розмірів — але є галузеві стандарти яких дотримується більшість. Знаючи їх — ти відразу розумієш макет і не витрачаєш час на підбір.

Ширини контейнерів — що і коли
Ширина Використання Приклад коду
max-width: 1440px Маркетингові лендінги, wide layout. Великий простір для фонів і декору. Stripe, Linear, Apple
max-width: 1280px Стандартний десктоп контейнер. Добре виглядає на 1366px і 1920px екранах. Bootstrap .container-xl

Дерево рішень — яку одиницю обрати
Це текст / шрифт?
→ rem для base text, clamp()+vw для заголовків, em для padding кнопок
Це рамка, тінь, або піксельно-точний елемент?
→ px завжди. border, box-shadow, icon size — завжди пікселі
Це ширина контейнера/колонки?
→ % + max-width, або fr в Grid, або min(100% - 48px, 1200px)
Це висота секції на весь екран?
→ 100dvh (мобільний сумісний) або 100vh (якщо тільки десктоп)
Хочу щоб розмір плавно змінювався між мін і макс?
→ clamp(мін, fluid, макс) — замінює min-width + max-width + fluid значення
Це ширина параграфа / блоку тексту?
→ max-width: 65ch — оптимальна читабельність незалежно від шрифту
Головне правило: спочатку не задавай жодних розмірів — подивись як виглядає без них. Браузер робить багато правильно за замовчуванням. Додавай розміри тільки тоді коли є конкретна причина.
