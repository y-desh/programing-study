01 — Про курс

Що ми будуємо і навіщо це все

Ed Row бачить що початківці витрачають надто багато часу на механічне запам'ятовування CSS властивостей, застрягають на крайніх випадках і не переходять до реального будування. Мета цього курсу — пропустити весь цей шум і зосередитись на тому CSS що справді потрібен.

Два проєкти — весь необхідний CSS
ПРОЄКТ 1 — Лекції 01–07
Освітній лендінг
Простий і елегантний. Ти вивчаєш: селектори, типографіку, кольори, box model, flexbox, форми, фонові зображення — базу яка покриває 80% реальних задач.
ПРОЄКТ 2 — Лекції 08–10
Новинний сайт
Складні layouts. Коли flex, коли grid, коли position. Grid template areas, overlay через ::before/::after, адаптивний дизайн через media queries.

«My goal is to make sure you skip all that noise and focus on the most essential aspects of CSS — get you to a point where you can actually start building applications.»

— ED ROW
02 — Налаштування

VS Code, Live Server, Prettier

Перш ніж писати код — налаштовуємо середовище. Ed Row використовує три інструменти.

Три інструменти для роботи
1
Visual Studio Code — редактор коду
Найпопулярніший редактор для веб-розробки. Скачай на code.visualstudio.com для свого OS. Безкоштовний.
2
Live Server — автооновлення браузера
Розширення VS Code. Extensions (Ctrl+Shift+X) → шукай "Live Server" → Install. З'явиться кнопка "Go Live" в правому нижньому куті. Натисни — відкриє сайт на localhost:5500 і автоматично оновлюватиметься при кожному збереженні файлу.
3
Prettier — форматування коду (опціонально)
Extensions → "Prettier - Code formatter" → Install. Автоматично форматує код при Ctrl+S. Ed Row бачиш це у відео — код сам вирівнюється при збереженні.

03 — Анатомія CSS

Як читати CSS правило

Ed Row пояснює: HTML — це структура будинку (стіни, кімнати). CSS — фарба, меблі, декор. Разом вони створюють завершений і функціональний будинок. CSS вибирає HTML елементи через селектори.

Анатомія CSS правила
/*  СЕЛЕКТОР    ВЛАСТИВІСТЬ      ЗНАЧЕННЯ   */
    .hero      {   color        :   red         ;   }
/*             ↑                             ↑
           відкриває блок               крапка з комою
           декларацій                   завершує кожну
                                        декларацію      */

/* Кілька декларацій в одному правилі */
.hero {
  color: white;
  background-color: #333;
  padding: 100px;
  /* коментар — браузер ігнорує */
}
Крапка з комою обов'язкова Кожна декларація завершується крапкою з комою. Без неї браузер може не розпізнати наступну властивість. Звичка ставити її завжди врятує від дивних багів.
04 — Selectors

Чотири типи селекторів

Ed Row: "There are four main types of selectors. If you know these, you know the majority of selectors." Решта — опціонально, вивчиш по ходу.

1 · Universal Selector — всі елементи одразу
/* * вибирає АБСОЛЮТНО ВСЕ на сторінці */
* {
  padding: 0;
  margin: 0;
  box-sizing: border-box;
}

/* Це перший рядок кожного CSS файлу.
   Скидає браузерні дефолти.
   Використовується ОДИН РАЗ.          */

/* Також: */
* {
  background-color: yellow; /* все жовте */
}
/* але зазвичай тільки reset */
Навіщо скидати margin і padding? Кожен браузер (Chrome, Firefox, Safari) додає свої відступи за замовчуванням. Вони різні. Universal reset робить однакову базу в усіх браузерах.
box-sizing: border-box Без цього — padding збільшує реальний розмір елемента. З border-box — padding включається в width. Детально в лекції Box Model.
2 · Element Selector — по HTML тегу
/* Вибирає ВСІ елементи цього тегу */
body {
  background-color: #f7f7f7;
  color: #333;
  /* font-family успадкується на все */
  font-family: Poppins, Times, serif;
}

div  { color: blue; }   /* всі div */
h1   { font-size: 2rem; }
p    { line-height: 1.6; }
header { background: #111; }
Де використовувати body — глобальний шрифт і базовий колір. h1–h6 — базові розміри. a — колір посилань. Загальні базові стилі що застосовуються до ВСІХ елементів типу.
Небезпека Якщо є 50 div на сторінці — всі отримають blue. Якщо хочеш стилізувати конкретні div — використовуй class.
3 · Class Selector — 90% твого CSS ★
index.html
<!-- class="..." — атрибут HTML -->
<header class="hero">
  <h1 class="hero__title">
    Empower Young Minds
  </h1>
  <p class="hero__description">
    <span class="hero__highlight">
      Innovative
    </span>
    educational platform
  </p>
</header>

<!-- Кілька класів на одному елементі -->
<h2 class="feature__title
           feature__title--selected">
  Empower Student Learning
</h2>
styles.css
/* . (крапка) перед назвою */

.hero {
  padding: 100px;
}

.hero__title {
  color: black;
  font-size: 2.2rem;
}

.hero__description {
  color: #81da76; /* жовтуватий */
}

.hero__highlight {
  color: #eee; /* майже білий */
}

/* Другий клас — додатковий стиль */
.feature__title--selected {
  text-decoration: underline;
}
Чому НЕ ID? ID може бути лише один на сторінці. Class можна використати скільки завгодно разів. Коли захочеш застосувати однаковий стиль до кількох елементів — з ID не вийде, доведеться переробляти. Ed Row: "I generally see no problems with just using classes everywhere."
4 · ID Selector — майже не використовується
index.html
<!-- id="..." — унікальний для сторінки -->
<span id="hero-highlight">
  Innovative
</span>
styles.css
/* # (решітка) перед назвою */
#hero-highlight {
  color: white;
}
/* Якщо хочеш ще раз цей стиль —
   доведеться рефакторити в class */
Де ID справді використовується Переважно в JavaScript: document.getElementById('hero-highlight'). Для CSS — застарілий підхід. Також ID має вищу специфічність ніж class, що може спричинити несподівані переваги стилів.
05 — Успадкування

Батьківсько-дочірні відносини

Це фундаментальна концепція CSS. Ed Row акцентує: "This is a very important relationship to really understand." Елементи вкладені один в одний утворюють дерево. Деякі CSS властивості передаються від батька до всіх дітей автоматично.

Приклад з проєкту — feature секція
<section class="feature">         ← батько
  <div class="feature__container">  ← дитина
    <div class="feature__item">     ← онук
      <h2 class="feature__title">
        Empower Students
      </h2>
      <p class="feature__description">
        Опис...
      </p>
    </div>
  </div>
</section>
/* Якщо встановити колір на батькові: */
.feature {
  color: blue-violet;
  /* ↓ успадковується на:
     feature__container,
     feature__item,
     feature__title,
     feature__description —
     всі стануть blue-violet!  */
}

/* Але padding/margin НЕ успадковуються */
.feature {
  padding: 100px;
  /* тільки .feature має 100px padding,
     дочірні елементи — ні             */
}
Що успадковується color, font-family, font-size, font-weight, line-height, letter-spacing, text-align — текстові властивості. Решта (padding, margin, border, background) — ні.
Практична демонстрація
section.feature (батько)
div.feature__container (дитина)
div.feature__item (онук)
Empower Students
Опис особливостей платформи
↑ Натисни кнопку — колір змінюється на батькові (.feature), але успадковується всіма дочірніми елементами автоматично. Саме тому font-family ставлять на body — весь сайт отримує шрифт.
06 — Старт проєкту

Перші рядки CSS для лендінгу

Ed Row починає з конкретного коду. Ось що він пише першим для Проєкту 1.

Перший CSS файл — styles.css
styles.css
/* ── SECTION: Selectors ────────────────────────── */

/* Universal reset — завжди перше */
* {
  padding: 0;
  margin: 0;
  box-sizing: border-box;
}

/* Body — глобальні базові стилі */
body {
  background-color: #f7f7f7; /* трохи сірий, не чистий білий */
  color: #333;
}

/* Hero section */
.hero {
  padding: 100px;
}

/* Перший текст */
.hero__title {
  color: black;
}

.hero__description {
  color: #81da76; /* жовто-зелений */
}

.hero__highlight {
  color: white; /* Innovative — білий на темному */
}
Чому #f7f7f7 а не white? Ed Row: "Generally for any website you generally don't want to use straight white because it's a little jarring. You generally want to use a slightly off color." Трохи темніший за білий — приємніший для очей.
🎯 Завдання — Лекція 01
1. Структура: Створи папку project/ з index.html і styles.css в одній директорії
2. HTML: В index.html зроби <header class="hero"> з <h1 class="hero__title">Empower Young Minds</h1>
3. CSS reset: Напиши universal reset (* padding:0, margin:0, box-sizing:border-box) і base body стилі
4. Класи: Застосуй .hero { padding: 100px } і .hero__title { color: black }
5. Перевір: Запусти Live Server, переконайся що все відображається
6. Поекспериментуй: Поміняй color у .hero — подивись чи успадковується h1
Підказка: якщо не бачиш padding — можливо body/html мають дефолтний margin. Universal reset вирішує це.
