01 — font-family

Підключення шрифтів

Ed Row використовує Google Fonts — безкоштовний сервіс від Google з сотнями шрифтів. Для проєкту обрано Poppins. Підключається через <link> в HTML, далі використовується через font-family в CSS.

Як підключити Google Fonts
index.html → <head>
<head>
  <!-- Підключення Google Fonts (Poppins) -->
  <link
    href="https://fonts.googleapis.com/
    css2?family=Poppins:wght@300;400;
    500;600;700&display=swap"
    rel="stylesheet"
  >

  <!-- Наш CSS ПІСЛЯ Google Fonts -->
  <link rel="stylesheet"
        href="styles.css">
</head>
wght@300;400;500;600;700 Завантажуємо тільки потрібні насиченості. Що менше — то швидший сайт. Не завантажуй всі якщо не потрібно.
styles.css
body {
  /* Перший = основний шрифт  */
  /* Решта = fallback шрифти  */
  font-family: Poppins, Times, serif;

  /* Якщо Poppins не завантажиться
     (нема інтернету або шрифт
     заблокований) — браузер
     спробує Times, потім serif  */
}

/* ЧОМУ в body, а не в .hero?   
   Через успадкування!           
   body — батько всієї сторінки. 
   Всі елементи отримають Poppins. */
generic fallback serif, sans-serif, monospace — вбудовані в браузер "групи шрифтів". Завжди ставь як останній fallback.
Де взяти шрифти — fonts.google.com
1. Заходиш на fonts.google.com
2. Шукаєш шрифт (наприклад Poppins)
3. Обираєш потрібні насиченості (Select styles)
4. Натискаєш "Get embed code" і копіюєш <link>
5. Вставляєш у <head> перед своїм styles.css
6. В CSS пишеш: font-family: 'Poppins', sans-serif;
02 — font-size

px vs rem — яку одиницю використовувати

Це одна з найважливіших тем. Ed Row пояснює різницю між абсолютними (px) і відносними (rem) одиницями і чому rem — кращий вибір для шрифтів.

Порівняння px і rem
px — абсолютна одиниця
.feature__title {
  font-size: 32px;
}

/* Завжди 32px на всіх екранах.
   Не залежить ні від чого.
   Якщо юзер збільшив шрифт
   в браузері — px НЕ адаптується */
Коли використовувати px Border (1px solid), box-shadow, icon size — де потрібна точна пікселізація. Для шрифтів — краще rem.
rem — відносна одиниця ★
.feature__title {
  font-size: 2.2rem;
}

/* 1rem = font-size елемента html.
   Зазвичай: 1rem = 16px (дефолт).
   
   2.2rem = 2.2 × 16px = 35.2px
   
   Якщо юзер змінив розмір шрифту
   в браузері — rem адаптується!  */
Чому rem кращий Масштабується з налаштуваннями доступності браузера. Людям з поганим зором що збільшили системний шрифт — ваш текст також збільшиться.

03 — font-weight

Насиченість шрифту

font-weight визначає товщину шрифту. Ed Row демонструє різні значення і каже що 500 — гарне для заголовків проєкту.

Шкала font-weight
.feature__title {
  font-weight: 500; /* середній — Ed Row вибрав для feature title */
}

/* Числові значення: */
font-weight: 100;  /* Thin — надтонкий */
font-weight: 200;  /* Extra Light      */
font-weight: 300;  /* Light            */
font-weight: 400;  /* Regular — нормальний (default) */
font-weight: 500;  /* Medium           */
font-weight: 600;  /* Semi Bold        */
font-weight: 700;  /* Bold             */
font-weight: 800;  /* Extra Bold       */
font-weight: 900;  /* Black — максимум */

/* Словесні значення (менш точні): */
font-weight: normal;  /* = 400 */
font-weight: bold;    /* = 700 */
200 — Extra Light — Empower Young Minds
300 — Light — Empower Young Minds
400 — Regular — Empower Young Minds
500 — Medium — Empower Young Minds ← Ed Row
600 — Semi Bold — Empower Young Minds
700 — Bold — Empower Young Minds
900 — Black — Empower Young Minds
Важливо: потрібно підключити насиченості в Google Fonts Якщо не підключив wght@500 — браузер буде автоматично генерувати "фейковий" bold і виглядатиме гірше. Підключуй тільки ті насиченості що використовуєш.
04 — Текстові властивості

text-transform, margin, text-align, line-height, letter-spacing

Ed Row швидко проходить через текстові властивості для .feature__description. Більшість із них не настільки поширені але корисно знати.

CSS для feature секції з проєкту
styles.css — feature section
/* ── SECTION: Fonts ─────────────────────────────── */

.feature {
  font-family: Poppins, Times, serif;
  /* В body краще — все успадкує */
  padding: 100px;
}

.feature__item {
  padding: 30px 0; /* 30px зверху і знизу, 0 ліво/право */
}

.feature__title {
  font-size: 2.2rem;
  font-weight: 500;
  text-transform: uppercase; /* ВЕЛИКІ ЛІТЕРИ через CSS */
  color: #333;
  margin-bottom: 10px;        /* відступ знизу */
}

.feature__title--selected {
  /* Другий клас — тільки для виділеного */
  text-decoration: underline;
}

.feature__description {
  /* text-align: центрування тексту */
  text-align: center;     /* left | center | right */

  /* line-height: відстань між рядками */
  line-height: 1.6;        /* ЧИСЛО без одиниць! = 1.6 × font-size */
  /* 2.4rem теж можна, але число — краще */

  /* letter-spacing: відстань між літерами */
  letter-spacing: 0.5px;  /* рідко потрібно */

  /* Колір параграфу */
  color: #666;
}

margin vs padding — різниця
padding — всередині коробки
.feature__item {
  padding: 30px 0;
  /* 30px зверху і знизу
     0 ліво і право
     
     Це ЧАСТИНА елемента.
     Колір фону заповнює padding. */
}
margin — зовні коробки
.feature__title {
  margin-bottom: 10px;
  /* Відстань між title і
     description — між
     двома різними коробками.
     
     НЕ частина елемента.
     Колір фону не заповнює. */
}

🎯 Завдання — Лекція 02
1. Підключи Google Fonts (Poppins або будь-який інший) до свого index.html
2. Перемісти font-family з .hero до body — перевір що всі елементи успадкують шрифт
3. Для .feature__title встанови: font-size 2.2rem, font-weight 500, text-transform uppercase
4. Для .feature__description встанови: text-align center, line-height 1.6, color #666
5. Поекспериментуй: зміни font-weight від 200 до 900 — поспостерігай різницю
6. Порівняй line-height 1.2 vs 1.6 vs 2.4 — відчуй як текст "дихає"
Підказка: якщо шрифт не змінився — перевір чи правильно підключено <link> в <head> і чи збігається назва в font-family
