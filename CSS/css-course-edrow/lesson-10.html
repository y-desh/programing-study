01 — position: absolute/relative

Тільки для overlay — не для layout

Ed Row демонструє чому position absolute НЕ підходить для layout і показує єдиний правильний випадок — overlay.

Чому position НЕ для layout
/* ❌ НЕПРАВИЛЬНО — position для layout */
#test  { position: absolute; right: 5px; }
#test2 { position: absolute; right: 200px; }

/* Проблема 1: при звуженні екрану елементи
   перекриваються — не адаптуються              */

/* Проблема 2: видали .gallery з навбару —
   .articles не займає його місце, залишається
   на фіксованій позиції                        */

/* ✅ ПРАВИЛЬНО — flex для layout */
.header__nav {
  display: flex;
  justify-content: space-between;
  /* При видаленні елемента — решта
     автоматично перебудовується ✅  */
}
Ed Row прямо каже: "I want to briefly go over why position absolute is not ideal for layout. Generally if you just remove Gallery — everything else aligns together automatically with Flex. With position absolute — it stays there, which is not desirable."
✅ Правильний кейс: текст поверх зображення
<!-- HTML: батько + дитина -->
<div class="hot-card">
  <img src="news.jpg">
  <div class="hot-card__overlay">
    Election 2024
  </div>
</div>

/* CSS */
.hot-card {
  position: relative; ← батько
}

.hot-card__overlay {
  position: absolute; ← дитина
  bottom: 0;
  left: 0;
  right: 0;
  padding: 1rem;
  background: rgba(0,0,0,0.7);
  color: white;
  font-weight: bold;
}
Election 2024 results...
relative батько + absolute дитина
Ключове правило: position: relative на батькові = точка відліку. position: absolute на дитині = позиціонування відносно батька, а не сторінки. Без relative на батькові — absolute буде відраховувати від найближчого positioned ancestor або сторінки.
02 — ::before і ::after

Псевдоелементи для hover overlay

Ed Row показує спосіб зробити hover overlay БЕЗ додаткового HTML — через ::before і ::after псевдоелементи.

Hover overlay через ::before/::after
.masonry-item {
  position: relative; ← батько
}

/* ::before — темний оверлей */
.masonry-item::before {
  content: "";         ← обов'язково!
  position: absolute;
  top: 0; left: 0;
  right: 0; bottom: 0;
  background: rgba(0,0,0,0.5);
  opacity: 0;           ← прихований
  transition: opacity 0.3s ease;
}

/* При hover — показуємо */
.masonry-item:hover::before {
  opacity: 1;
}

/* ::after — текст "View Story" */
.masonry-item::after {
  content: "View Story";
  position: absolute;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%);
  /* centering трюк через translate */
  color: white;
  font-weight: bold;
  font-size: 1.5rem;
  opacity: 0;
  transition: opacity 0.3s ease;
}
.masonry-item:hover::after {
  opacity: 1;
}

03 — Media Queries

@media — адаптивний дизайн

Ed Row: "If you did it correctly — this will be super easy. That's why I stress this foundational layout principle." Правильний flex/grid layout → мінімальні media queries.

@media синтаксис і практика App 2
/* Базові стилі — для десктопу */
.header__links { display: flex; gap: 2rem; }
.featured { display: flex; height: 600px; }
.masonry-grid { display: grid; grid-template-columns: repeat(5,1fr); }

/* При ширині <= 768px — мобільна версія */
@media screen and (max-width: 768px) {

  /* Ховаємо навбар посилання */
  .header__links {
    display: none;
    /* (у реальному — замінили б на меню-іконку) */
  }

  /* Featured: горизонталь → вертикаль */
  .featured {
    flex-direction: column;
    height: auto;
    gap: 2rem;
  }

  /* Grid → block (все в одну колонку) */
  .masonry-grid {
    display: block;
  }
  .masonry-item {
    height: 300px;
    margin-bottom: 1rem;
  }

  /* Container менший padding */
  .container {
    padding: 2rem 1rem;
  }
}
Чому це так просто? Ed Row: "If you did it correctly — this will be super easy. So just two lines to make grid become block for responsive. This handles the majority of cases." Правильний layout = легкий responsive.
max-width vs min-width — desktop-first vs mobile-first
/* max-width: 768px — Desktop First (Ed Row)   
   Базові стилі = десктоп                       
   @media = зміни для мобільного               */
@media (max-width: 768px) { /* при <= 768px */
  /* мобільні зміни */
}

/* min-width: 768px — Mobile First             
   Базові стилі = мобільний                    
   @media = розширення для десктопу            
   ← Сучасний підхід, рекомендований          */
@media (min-width: 768px) { /* при >= 768px */
  /* десктопні розширення */
}

/* Типові breakpoints: */
/* 480px  — дуже маленький телефон */
/* 768px  — планшет (iPad)         */
/* 1024px — ноутбук                */
/* 1200px — десктоп                */
/* 1500px — великий монітор        */
