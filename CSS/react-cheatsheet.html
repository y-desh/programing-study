Основні концепції

Компонент
Функція яка повертає JSX. Це і є React. Ім'я з великої літери.
Props
Дані що передаються в компонент ззовні. Тільки читання — не змінюй props.
State
Внутрішні дані компонента. При зміні — React перерендерює компонент.
Мінімальний компонент
// MyComponent.jsx
export default function MyComponent({ title, count }) {
  return (
    <div>
      <h1>{title}</h1>
      <p>Count: {count}</p>
    </div>
  )
}
React — це UI бібліотека, не фреймворк. Відповідає тільки за рендеринг. Routing, state management, fetching — окремі бібліотеки.

JSX — правила і нюанси

Ключові правила JSX
Правило❌ Неправильно✓ Правильно
Один кореневий елемент <h1/><p/> <><h1/><p/></>
class → className class="btn" className="btn"
for → htmlFor for="email" htmlFor="email"
Самозакривний тег <img> <img />
camelCase атрибути onclick, tabindex onClick, tabIndex
Інлайн стилі — об'єкт style="color: red" style={{ color: 'red' }}
JS у фігурних дужках "Hello " + name {"Hello " + name}
Fragment — коли не потрібен зайвий div
// Обидва варіанти однакові
return <React.Fragment>...content...</React.Fragment>
return <>...content...</>  // коротший синтаксис
Template literals у JSX
// ❌ Неправильно
<div className="btn {isActive ? 'active' : ''}">

// ✓ Правильно — template literal у фігурних дужках
<div className={`btn ${isActive ? 'active' : ''}`}>

Компоненти і Props

Props — передача даних
// Батьківський компонент
function App() {
  return <Button label="Click me" disabled={false} onClick={handleClick} />
}

// Дочірній компонент — деструктуризація props
function Button({ label, disabled = false, onClick }) {
  return <button disabled={disabled} onClick={onClick}>{label}</button>
}

// Props зі spread оператором
const btnProps = { label: 'Save', disabled: true }
return <Button {...btnProps} />
children — вміст між тегами
function Card({ children, title }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      {children}  // все що між <Card> ... </Card>
    </div>
  )
}

// Використання
<Card title="Profile">
  <p>Будь-який JSX тут стане children</p>
</Card>
Props передаються тільки вниз (від батька до дитини). Для зворотної передачі даних — передавай функцію як prop.

Хуки — повний огляд

useState
const [val, setVal] = useState(0)
Локальний стан компонента. При виклику setter — компонент перерендерюється.
const [count, setCount] = useState(0)
const [user, setUser] = useState(null)
const [items, setItems] = useState([])
useEffect
useEffect(() => {}, [deps])
Побічні ефекти: fetch, підписки, маніпуляції DOM. Залежності у масиві контролюють коли запускається.
useEffect(() => { /* запуск при mount і кожному render */ })
useEffect(() => { /* тільки при mount */ }, [])
useEffect(() => { /* при зміні id */ }, [id])
useEffect(() => { return () => { /* cleanup при unmount */ } }, [])
useRef
const ref = useRef(null)
Доступ до DOM елементу або зберігання значення без ре-рендеру. ref.current — завжди актуальне значення.
const inputRef = useRef(null)
<input ref={inputRef} />
inputRef.current.focus()  // прямий доступ до DOM
useMemo
useMemo(() => val, [deps])
Кешує результат важкого обчислення. Перераховується тільки коли змінились залежності.
const sorted = useMemo(() =>
  [...items].sort((a, b) => a.name - b.name),
  [items]  // перераховує тільки при зміні items
)
useCallback
useCallback(fn, [deps])
Кешує функцію між рендерами. Потрібен коли передаєш функцію у мемоізований дочірній компонент.
const handleClick = useCallback(() => {
  doSomething(id)
}, [id])  // нова функція тільки при зміні id
useContext
useContext(MyContext)
Читає значення з Context без prop drilling. Компонент підписується на зміни контексту.
const ThemeContext = createContext('light')
const theme = useContext(ThemeContext)
useReducer
useReducer(reducer, init)
Альтернатива useState для складного стану з кількома пов'язаними значеннями.
const [state, dispatch] = useReducer(reducer, { count: 0 })
dispatch({ type: 'increment' })
Хуки — тільки всередині функціонального компонента або кастомного хука. Ніколи в умовах, циклах або вкладених функціях.
🔄

State — правила роботи

✓ Правильно — новий об'єкт
// Масив — завжди новий масив setItems([...items, newItem]) setItems(items.filter(i => i.id !== id)) setItems(items.map(i => i.id === id ? {...i, done: true} : i )) // Об'єкт — spread для копії setUser({...user, name: 'Новий'}) // Функціональне оновлення setCount(prev => prev + 1)
❌ Неправильно — мутація
// Пряма мутація масиву items.push(newItem) setItems(items) // не перерендерить! // Пряма мутація об'єкту user.name = 'Новий' setUser(user) // та сама референція! // Без функціонального оновлення setCount(count + 1) // stale closure setCount(count + 1) // +1 замість +2
State batching — React групує оновлення
// React оновить DOM тільки один раз після обох setstate
function handleClick() {
  setCount(c => c + 1)
  setName('новий')
  // один ре-рендер, не два
}
State оновлюється асинхронно. Одразу після setState новий стан недоступний — він буде при наступному рендері.

useEffect — fetch і підписки

Fetch даних — правильний патерн
function UserProfile({ userId }) {
  const [user, setUser] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    let cancelled = false  // захист від race condition

    setLoading(true)
    fetch(`/api/users/${userId}`)
      .then(r => r.json())
      .then(data => { if (!cancelled) setUser(data) })
      .catch(err => { if (!cancelled) setError(err) })
      .finally(() => { if (!cancelled) setLoading(false) })

    return () => { cancelled = true }  // cleanup
  }, [userId])  // перезапуск при зміні userId

  if (loading) return <p>Loading...</p>
  if (error) return <p>Error: {error.message}</p>
  return <div>{user.name}</div>
}
Нескінченний цикл — якщо у залежностях об'єкт або масив що створюється при рендері. Він завжди "новий" навіть з тими самими даними.
📋

Списки і умовний рендеринг

Рендеринг списку — завжди key
// ✓ key — унікальний і стабільний ідентифікатор
{items.map(item => (
  <Card key={item.id} title={item.title} />
))}

// ❌ Не використовуй index як key (якщо список змінюється)
{items.map((item, index) => (
  <Card key={index} />  // баги при сортуванні/видаленні
))}
Умовний рендеринг — три способи
// 1. Тернарний оператор
{isLoggedIn ? <Dashboard /> : <Login />}

// 2. && оператор (тільки коли показати або нічого)
{isAdmin && <AdminPanel />}

// ⚠ Небезпека з &&: якщо count=0, відрендериться "0"
{count && <Badge />}       // ❌ виведе 0
{count > 0 && <Badge />}  // ✓ правильно

// 3. Ранній return
if (!data) return <Spinner />
return <Content data={data} />

Events — обробка подій

Основні патерни
// ✓ Передача функції (не виклик)
<button onClick={handleClick}>     // правильно
<button onClick={handleClick()}>   // ❌ викличеться одразу

// Передача аргументу — через arrow function
<button onClick={() => handleDelete(item.id)}>

// Доступ до event об'єкта
<input onChange={e => setValue(e.target.value)}>

// preventDefault
function handleSubmit(e) {
  e.preventDefault()
  // обробка форми
}
<form onSubmit={handleSubmit}>
Контрольований input
const [value, setValue] = useState('')

<input
  value={value}
  onChange={e => setValue(e.target.value)}
/>

// ⚠ Без onChange React покаже warning і input буде readonly

Поширені патерни

Custom Hook
Reusability 
// useFetch.js
function useFetch(url) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    fetch(url)
      .then(r => r.json())
      .then(setData)
      .finally(() => setLoading(false))
  }, [url])

  return { data, loading }
}

// Використання
const { data, loading } = useFetch('/api/users')
Context + Provider
Global State 
const ThemeContext = createContext()

function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light')

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}

// Читання у будь-якому дочірньому
const { theme } = useContext(ThemeContext)
Compound Components
Composition 
// Компоненти що працюють разом
<Select value={val} onChange={setVal}>
  <Select.Option value="a">Option A</Select.Option>
  <Select.Option value="b">Option B</Select.Option>
</Select>

// Реалізація через static methods
Select.Option = function({ value, children }) {
  return <option value={value}>{children}</option>
}
Lifting State Up
State Sharing 
// Коли двом компонентам потрібен один стан
// — виноси state у спільного батька
function Parent() {
  const [value, setValue] = useState('')

  return (
    <>
      <Input value={value} onChange={setValue} />
      <Preview value={value} />
    </>
  )
}
Lazy loading — код-сплітинг
import { lazy, Suspense } from 'react'

// Завантажується тільки коли потрібен
const Dashboard = lazy(() => import('./pages/Dashboard'))

function App() {
  return (
    <Suspense fallback={<Spinner />}>
      <Dashboard />
    </Suspense>
  )
}

Типові помилки початківців

!
Нескінченний useEffect
Об'єкт або масив у залежностях створюється при кожному рендері — нова референція = нескінченний цикл. Рішення: useMemo або винести за межі компонента.
!
Мутація state напряму
items.push(x) або user.name = 'new' — React не побачить зміни і не перерендерить. Завжди створюй новий об'єкт/масив.
!
Виклик хука з умовою
if (x) { useState() } — порушення Rules of Hooks. Хуки завжди викликаються в одному і тому самому порядку.
!
Stale closure у useEffect
Читаєш state всередині useEffect але не додав його у залежності — отримуєш старе значення. Додавай всі змінні що використовуєш у масив залежностей.
!
Key як index при динамічному списку
key={index} при сортуванні або видаленні призводить до багів з анімаціями і state. Завжди використовуй унікальний id елементу.
!
Занадто великі компоненти
Якщо компонент більше 150 рядків — розбий на менші. Один компонент = одна відповідальність.
!
Забутий cleanup у useEffect
Підписки, таймери, fetch запити потребують cleanup функції — інакше memory leak або update на unmounted компонент.
Швидкий чекліст перед пушем
✓Всі хуки на верхньому рівні компонента
✓useEffect має всі залежності у масиві
✓State не мутується напряму
✓Списки мають key з id, не index
✓Обробники — функції, не виклики
✓Контрольовані inputs мають onChange
✓Компоненти з великої літери
✓Props не змінюються всередині компонента
