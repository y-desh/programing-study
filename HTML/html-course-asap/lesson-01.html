01 — Що таке HTML

HTML, CSS і JavaScript — різниця

Перш ніж писати код — треба зрозуміти навіщо він взагалі існує. ASAP Frontend пояснює просто і чітко через реальний приклад з їх сайту.

Три технології веб-розробки
🏗️
HTML
Що є на сторінці
Markup Language — не програмна мова. Визначає вміст: текст, зображення, кнопки, форми. Вся видима структура.
🎨
CSS
Як виглядає
Стилі HTML елементів: кольори, шрифти, розміри, відступи, анімації. Без CSS — сайт виглядає як текстовий документ.
⚡
JavaScript
Логіка та правила
Програмна мова. Якщо користувач залогінений — показуємо відео. Якщо ні — показуємо popup. Інтерактивність.
Реальний приклад з ASAP Frontend
Відкрий будь-який сайт → правий клік → Inspect → вкладка Elements. Весь код який ти бачиш — це HTML. Він визначає що є на сторінці. 
<!-- Це HTML. Він кажe ЩО є на сторінці -->
<img>   ← це зображення
<button>Купити</button>   ← це кнопка
<p>Текст статті</p>   ← це параграф

/* Це CSS. Він каже ЯК ВИГЛЯДАЄ */
button { background: blue; color: white; }

// Це JavaScript. Він каже ЩО ВІДБУВАЄТЬСЯ
if (userLoggedIn) { showVideo() } else { showPopup() }

«HTML is not a programming language. Instead, it's a markup language that we use to build web pages.»

— ASAP FRONTEND
02 — Setup

VS Code і три розширення

ASAP Frontend використовує VS Code з трьома розширеннями. Встанови все це перед тим як починати — заощадить час.

Крок 1 · VS Code — редактор коду
Йди на code.visualstudio.com → завантаж для своєї ОС → встанови. VS Code — найпопулярніший редактор для веб-розробки, безкоштовний від Microsoft.
Для macOS (твій випадок) Скачай .dmg файл → перетягни VS Code в Applications → запусти.
Відкрити розширення:
Mac: Cmd + Shift + X
Win: Ctrl + Shift + X

Або через іконку Extensions на лівій панелі.
Крок 2 · Три обов'язкових розширення
Prettier
Prettier — Code formatter
Автоматично форматує код. Натиснув Alt+Shift+F (Mac: Option+Shift+F) — і код став акуратним. 46+ млн завантажень.
Live Server
Live Server
Запускає локальний сервер. Правий клік на HTML файлі → Open with Live Server. При збереженні — браузер автоматично оновлюється. 50+ млн завантажень.
Emmet
Emmet (вбудований або Mithil Emmet)
Скорочення для HTML. Напишіть ! → Enter — і отримаєте повний HTML boilerplate. Або div → Enter — і відразу <div></div>.
03 — Перший файл

Створюємо index.html

Покрокова інструкція
1
Створи папку на робочому столі — назви html-crash-course
2
Відкрий VS Code → перетягни папку всередину (або File → Open Folder)
3
В Explorer (лівий бік) натисни іконку New File → назви index.html
4
Введи ! (знак оклику) → натисни Enter — Emmet генерує boilerplate
5
Правий клік на файлі → Open with Live Server → бачиш результат у браузері
Emmet boilerplate — що виходить після !
index.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

    <!-- Тут ти будеш писати HTML -->

</body>
</html>
Чому файл називається index.html? index.html — це домашня сторінка сайту за замовчуванням. Коли браузер відкриває папку — він шукає саме index.html першим.

Alt+Shift+F (Option+Shift+F на Mac) Prettier форматує код. Використовуй після кожного великого блоку — код стає читабельнішим.
04 — DevTools

F12 — твій найкращий друг

ASAP Frontend постійно використовує DevTools щоб показувати як браузер "бачить" HTML. Навчись відкривати це з першого дня.

Як відкрити і що там є
/* Відкрити DevTools: */
F12                    ← будь-який браузер
Ctrl+Shift+I           ← Windows/Linux
Cmd+Option+I           ← macOS

/* Або: */
Правий клік → Inspect

/* Вкладка Elements: */
← тут видно HTML дерево
← можна клікати на елементи
← можна редагувати прямо тут!

/* Редагування в DevTools: */
→ подвійний клік на тексті
→ зміни видно одразу
→ Refresh — повертає оригінал
Навіщо редагувати в DevTools? Тестуєш ідеї без зміни файлу. Якщо щось поламалось — просто оновив сторінку і все повернулось. Зміни в DevTools НЕ зберігаються.
Hover → побачиш розміри Наведи курсор на елемент в DevTools → браузер підсвітить його на сторінці і покаже розміри, padding, margin.
Elements panel щодня Відкривай DevTools коли щось виглядає не так. Це як рентген для твого HTML.
