HTML Tag Syntax
& Page Structure

Як влаштований HTML тег. Start/End теги, самозакриваючі теги. Структура сторінки: DOCTYPE, html, head, body. Різниця між тегом і елементом.

01 — HTML Теги

Анатомія HTML тегу

ASAP Frontend: "Everything in HTML is made up of tags. Different kinds of tags allow us to choose different kinds of content." Тег — основна одиниця HTML.

Структура тегу
<!-- Кутові дужки + назва тегу = тег -->
<h1>Текст заголовку</h1>
 ↑              ↑             ↑
Start tag    Вміст        End tag
(відкриває)  (content)   (закриває)
                          / = кінець

<p>Параграф тексту</p>
<button>Натисни мене</button>

<!-- Теги зазвичай йдуть ПАРАМИ:       -->
<!-- Start tag ... content ... End tag -->
<!-- End tag = те саме + forward slash -->
Ключові правила тегів 1. Кутові дужки < > навколо назви — обов'язково
2. Теги зазвичай парні: start і end
3. End tag = як start tag, але з / перед назвою
4. Вміст між тегами — те що бачить користувач
Тег vs Елемент — різниця
<!-- ТЕГ — це одна конструкція: -->
<h1>   ← це start TAG
</h1>  ← це end TAG

<!-- ЕЛЕМЕНТ — це start tag + вміст + end tag разом: -->
<h1>Hello World</h1>
 ↑                    ↑
 ←── увесь цей рядок = HTML ELEMENT ──→
Простіше запам'ятати так: Tag — кутовий дужкастий шматок (<h1>). Element — це тег + весь його вміст. В звичайній розмові їх часто плутають, але офіційно це різні речі.
Самозакриваючі теги (Self-closing)
<!-- Деякі теги НЕ мають end tag -->
<!-- Вони самозакриваються.       -->

<!-- Парний тег (звичайний): -->
<p>Текст</p>

<!-- Самозакриваючий тег: -->
<input type="text" />   ← / перед > — best practice
<img src="photo.jpg" alt="фото" />

<!-- Ще самозакриваючі теги: -->
<br />    ← перенос рядка
<hr />    ← горизонтальна лінія
<meta />  ← метадані
<link />  ← підключення CSS
Чому / перед закриваючою >? ASAP Frontend: "This helps us keep our code more consistent and clear. It makes very clear that this is not a start tag we forgot to close — it's a self-closing tag."
02 — Атрибути

Attributes — додаткова інформація про елемент

Атрибути — це спосіб дати елементу додаткову інформацію. Вони завжди в start tag і мають формат name="value".

Синтаксис атрибутів
<!-- Атрибут: ім'я="значення" -->
<a href="https://google.com">Google</a>
   ↑атрибут       ↑значення

<img src="photo.jpg" alt="Моє фото" />
      ↑1-й атр.         ↑2-й атрибут
      (де файл)         (опис для зрячих)

<input type="email" placeholder="Email..." />
       ↑тип поля          ↑підказка всередині

<!-- Кілька атрибутів — через пробіл -->
<a href="https://asapfrontend.com" target="_blank">
  ASAP Frontend
</a>
03 — Структура сторінки

DOCTYPE, html, head, body — кожен HTML документ

Кожна HTML сторінка має однакову базову структуру. Розберемо кожну частину детально.

Повна структура — з поясненнями
index.html
<!DOCTYPE html>
<!-- Каже браузеру: "це HTML5 документ".
     Завжди перший рядок. Завжди. -->

<html lang="uk">
<!-- Кореневий елемент.
     lang="uk" — мова сторінки (для SEO і reader'ів).
     Все HTML знаходиться всередині нього. -->

  <head>
    <!-- ГОЛОВА: метадані (невидима частина).
       Тут не буде НІЧОГО видимого на сторінці!
       Тут: підключення CSS, JS, налаштування. -->

    <meta charset="UTF-8" />
        <!-- Кодування символів. UTF-8 підтримує
         всі символи включно з кирилицею. -->

    <meta name="viewport"
          content="width=device-width, initial-scale=1.0" />
        <!-- Мобільна адаптація. Завжди включай. -->

    <title>HTML Crash Course</title>
        <!-- Заголовок вкладки в браузері.
         Також використовується пошуковиками. -->

  </head>

  <body>
    <!-- ТІЛО: весь видимий вміст сторінки.
       Текст, зображення, кнопки, форми —
       все сюди. Тут ти будеш проводити 95% часу. -->

    <h1>Привіт, HTML!</h1>
    <p>Мій перший параграф.</p>

  </body>

</html>
head vs body — що куди йде
<head> — невидима частина
✅ <title> — заголовок вкладки
✅ <meta> — кодування, viewport
✅ <link> — підключення CSS
✅ <script> — підключення JS
❌ Параграфи, заголовки, зображення
<body> — видима частина
✅ Заголовки (h1–h6)
✅ Параграфи (p)
✅ Зображення (img)
✅ Посилання (a), кнопки (button)
✅ Форми, списки, div'и...
Змінюємо title — бачимо результат
<!-- Зміни title -->
<title>Document</title>
   ↓ Зміни на:
<title>HTML Crash Course</title>

<!-- Збережи → бачиш у вкладці
     браузера: "HTML Crash Course" -->
HTML Crash Course

Hello World

I love ASAP Front End

↑ title відображається у вкладці
Коментарі в HTML
<!-- Це коментар. Браузер його не показує. -->

<!-- Ctrl+/ (Cmd+/) — ввімкнути/вимкнути коментар -->

<!-- Використовується для: -->
<!-- 1. Пояснення коду -->
<!-- 2. Тимчасове вимкнення коду -->
<!-- 3. Навігація по великому файлу -->

<h1>Заголовок</h1>
<!-- <p>Цей параграф вимкнений</p> -->
<p>Цей параграф активний</p>
🎯 Завдання — Лекція 02
1. В своєму index.html змінь <title> на "Мій HTML" → перевір у вкладці браузера
2. В <body> напиши <h1>Hello World</h1> і <p>Я вчу HTML</p>
3. Додай коментар перед h1: <!-- Головний заголовок -->
4. Спробуй Ctrl+/ (Cmd+/) щоб закоментувати h1 → перевір що він зник зі сторінки
5. Відкрий DevTools → Elements → знайди свій h1 і p
6. Виклич Alt+Shift+F (Option+Shift+F) → подивись як Prettier форматує код
Підказка: <!DOCTYPE html> завжди першим рядком. Якщо забудеш — сторінка може відображатись неправильно.
