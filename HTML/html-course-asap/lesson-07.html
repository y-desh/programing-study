Images
Зображення

Тег <img> — самозакриваючий. src: зовнішній URL vs локальний файл. Відносні шляхи і вкладені папки. alt атрибут для доступності.

01 — img tag

<img> — самозакриваючий тег

ASAP Frontend: "Just like an input element, this is a self-closing tag." img має два атрибути: src (де зображення) і alt (опис).

Базовий синтаксис
<!-- Images -->

<!-- img — самозакриваючий тег -->
<img src="URL або шлях" alt="опис" />
 ↑    ↑               ↑
 тег  де знайти     що це
      зображення    (для скрін-рідерів)

<!-- Emmet: img → Enter → отримуєш:
     <img src="" alt="" />            -->

<!-- Заповнюємо атрибути:    -->
<img src="superman.jpeg" alt="Superman image" />
img — inline елемент! За замовчуванням img — inline. Якщо хочеш щоб він займав весь рядок або мав відступи — обгорни в div (block). Через CSS можна поставити display: block на сам img.
02 — Зовнішній URL

Зображення з інтернету

Перший спосіб: вказати URL зображення з інтернету. Правий клік на будь-якому фото → Copy image address → вставляєш в src.

src — зовнішній URL
<!-- Спосіб 1: зовнішній URL -->
<!-- Правий клік на фото в браузері
     → Copy image address
     → вставляєш сюди:             -->

<img
  src="https://site.com/image.jpg"
  alt="Опис зображення"
/>

<!-- Наприклад ASAP Frontend logo: -->
<img
  src="https://asapfrontend.com/logo.png"
  alt="ASAP Frontend Logo"
/>

<!-- ✅ Переваги:
     Не треба завантажувати файл
     
     ❌ Недоліки:
     Якщо сайт впаде — зображення зникне
     Залежиш від зовнішнього сервера -->
Copy image address vs Copy image Copy image address — копіює URL. Copy image — завантажує файл. Нам потрібен URL → Copy image address.
Де знайти URL зображення? Google Images → клік на фото → правий клік → Copy image address. Або на будь-якому сайті — правий клік на фото.
03 — Локальний файл

Зображення зі свого комп'ютера

Другий спосіб: зберегти зображення в папку проєкту і вказати шлях до нього. HTML шукає файли відносно поточного HTML файлу.

Той самий рівень — найпростіший випадок
html-crash-course/
├── index.html
└── superman.jpeg   ← ПОРЯД з HTML!

<!-- Якщо файл поряд з HTML —
     просто пишемо ім'я файлу: -->
<img
  src="superman.jpeg"
  alt="Superman image"
/>

<!-- HTML починає пошук з папки
     де знаходиться index.html.
     Якщо superman.jpeg там же —
     просто пишемо ім'я!         -->
Чому це працює? ASAP Frontend: "HTML by default starts by searching for files on the same level as the current HTML file." Тобто в тій самій папці де лежить index.html.
Вкладені папки — шлях через /
Структура: один рівень вкладеності
html-crash-course/
├── index.html
└── assets/
    └── superman.jpeg

<!-- superman.jpeg тепер в assets/ -->
<!-- Потрібно вказати шлях:        -->
<img
  src="assets/superman.jpeg"
  alt="Superman"
/>

<!-- assets/ — заходимо в папку
     superman.jpeg — беремо файл -->
Як HTML шукає файл:
1. Старт з папки index.html
2. Бачить: assets/
3. Заходить всередину
4. Знаходить superman.jpeg ✅
Структура: два рівні вкладеності
html-crash-course/
├── index.html
└── assets/
    └── images/
        └── superman.jpeg

<!-- Потрібно пройти два рівні: -->
<img
  src="assets/images/superman.jpeg"
  alt="Superman"
/>

<!-- assets/ → images/ → superman.jpeg
     
     Кожен / = входимо в папку -->
Логіка шляху:
1. Старт з папки index.html
2. Бачить: assets/
3. Заходить всередину
4. Бачить: images/
5. Заходить всередину
6. Знаходить superman.jpeg ✅
VS Code — Compact Folders — вимкнути! ASAP Frontend показує: якщо папки мають одну вкладену папку — VS Code по дефолту згортає їх в один рядок. Щоб виправити: File → References and Settings → шукаємо "compact folders" → вимикаємо. Тепер структура папок зрозуміліша.
04 — alt атрибут

Доступність — alt для зображень

Навіщо alt і коли він показується
<img
  src="superman.jpeg"
  alt="Superman flying over city"
/>

<!-- alt показується/використовується:
     
     1. Якщо зображення не завантажилось
        (битий URL, нема інтернету)
        → браузер показує текст з alt
        
     2. Screen reader для людей з вадами зору
        → читає вголос текст alt
        
     3. SEO — пошуковики читають alt
        для розуміння що на фото         -->

<!-- Порожній alt — для декоративних фото: -->
<img src="background.jpg" alt="" />
<!-- Порожній alt = screen reader ігнорує -->
ASAP Frontend: "Let's say someone is blind and they are using what we call a screen reader — something that goes through your website and describes the content to the blind person. The alt attribute is where we tell the screen reader what our image is actually about."
Хороший alt текст Описуй ЩО на зображенні. "Superman flying over New York at night" — добре. "image1" або "photo" — погано.
Шпаргалка — src для різних ситуацій
Ситуаціяsrc
Файл в тій самій папціsrc="photo.jpg"
Файл в папці images/src="images/photo.jpg"
Файл в assets/img/src="assets/img/photo.jpg"
Зображення з інтернетуsrc="https://site.com/photo.jpg"
На один рівень вгоруsrc="../photo.jpg"
🎯 Завдання — Лекція 07
1. Знайди будь-яке зображення в Google Images → правий клік → Copy image address → встав в src
2. Завантаж зображення в папку проєкту → вкажи src з лише іменем файлу
3. Створи папку images/ → перемісти зображення туди → виправ src на images/назва.jpg
4. Для кожного img напиши осмислений alt текст
5. Зроби img що відкриває посилання при кліці: обгорни img в <a href="...">
6. Обгорни img в div → переконайся що div займає весь рядок (block)
Підказка: <a href="url"><img src="..." alt="..."></a> — клікабельне зображення
