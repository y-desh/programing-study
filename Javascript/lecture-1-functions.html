Функції в JavaScript

Що таке функції, як їх оголошувати, викликати, передавати параметри і повертати результат.

1 Що таке функція

Функція — це коробка з кодом. Ти пишеш код один раз, а викликаєш скільки завгодно разів з різними даними.

Аналогія: Уяви рецепт приготування кави. Рецепт написаний один раз, але ти можеш приготувати каву будь-коли — вранці, вдень, для друга. Функція — це рецепт. Виклик функції — це приготування кави. 
// Оголошення функції — пишемо рецепт
function greet() {
  console.log('Привіт!')
}

// Виклик функції — готуємо каву
greet()  // → Привіт!
greet()  // → Привіт!
greet()  // → Привіт! (можна скільки завгодно)

2 Параметри та аргументи

Параметри — це вхідні слоти функції. Аргументи — це конкретні значення які ти передаєш при виклику.

// name — параметр (змінна-слот)
function greet(name) {
  console.log('Привіт, ' + name + '!')
}

// 'Антон' і 'Марія' — аргументи (конкретні значення)
greet('Антон')  // → Привіт, Антон!
greet('Марія')  // → Привіт, Марія!

Функція може приймати кілька параметрів:

function add(a, b) {
  console.log(a + b)
}

add(2, 3)   // → 5
add(10, 20)  // → 30

3 Return — що повертає функція

return зупиняє функцію і повертає значення назовні. Без return функція повертає undefined.

Без return

function add(a, b) {
  a + b
  // нічого не повертає
}

const result = add(2, 3)
// result = undefined

З return

function add(a, b) {
  return a + b
  // повертає суму
}

const result = add(2, 3)
// result = 5
Аналогія: Ти попросив друга підрахувати суму. Без return — друг порахував, але не сказав тобі відповідь. З return — порахував і повідомив результат.

return також зупиняє виконання функції — код після нього не виконується:

function check(age) {
  if (age < 18) {
    return 'Неповнолітній'  // ← зупинились тут
  }
  return 'Повнолітній'      // ← сюди не дійшли якщо age < 18
}

check(15)  // → 'Неповнолітній'
check(25)  // → 'Повнолітній'

4 Виклик vs передача — найважливіше

Дужки () означають: виконай прямо зараз. Без дужок — це посилання на функцію, яку можна викликати пізніше.

function sayHi() {
  console.log('Hi!')
}

sayHi    // це сама функція (рецепт на папері)
sayHi()  // це виклик (приготування кави прямо зараз)

Чому це важливо? В React передаєш функцію в onClick — вона має виконатись при кліку, а не одразу:

❌ Виконується одразу

// Викликає при рендері!
onClick={deleteAll()}

✅ Виконується при кліку

// Передає функцію
onClick={deleteAll}
💡

Правило: якщо треба передати аргумент — потрібна стрілочна функція: onClick={() => deleteItem(id)}. Інакше deleteItem(id) виконається одразу.

5 Дефолтні параметри

Якщо аргумент не передали — можна задати значення за замовчуванням:

function greet(name = 'Гість') {
  console.log('Привіт, ' + name)
}

greet('Антон')  // → Привіт, Антон
greet()        // → Привіт, Гість (дефолт)

В React це використовується для props:

function Button({ variant = 'primary', size = 'md' }) {
  // якщо variant не передали — буде 'primary'
}

6 Скоуп — де живе змінна

Змінна оголошена всередині функції — існує тільки всередині неї:

function calculate() {
  const result = 42  // живе тільки тут
  console.log(result)  // → 42
}

calculate()
console.log(result)  // → ReferenceError! result не існує тут

Але функція бачить змінні з зовнішнього скоупу:

const name = 'Антон'  // зовнішня змінна

function greet() {
  console.log(name)  // → 'Антон' (бачить зовнішню)
}
Аналогія: Ти живеш в кімнаті в будинку. Ти бачиш що є в будинку (зовнішній скоуп). Але гості будинку не бачать що у тебе в кімнаті (внутрішній скоуп).
