Колбеки та стрілочні функції

Анонімні функції, стрілочний синтаксис, колбеки — і чому функції в JS передаються як значення.

1 Функція як значення

В JS функція — це звичайне значення. Її можна записати в змінну, передати в іншу функцію, повернути з функції.

// Функція — це значення, як число чи рядок
const num = 42
const text = 'hello'
const greet = function() { console.log('Hi!') }

// Можна передати в змінну і викликати
greet()  // → Hi!

// Можна передати в іншу функцію
setTimeout(greet, 1000)  // викличе greet через 1 секунду
В C# функції так не передаються — там є делегати і лямбди, але концепція менш природня. В JS це основа мови — функції є об'єктами першого класу.

2 Стрілочна функція

Коротший спосіб записати функцію. В React використовується майже скрізь.

function add(a, b) { return a + b }
Звичайна функція
→
(a, b) => a + b
Стрілочна функція

Різні форми запису — однаковий результат:

// З тілом — кілька рядків, потрібен return
const add = (a, b) => {
  const result = a + b
  return result
}

// Короткий запис — один вираз, return автоматичний
const add = (a, b) => a + b

// Один параметр — дужки можна прибрати
const double = n => n * 2

// Повертає обʼєкт — потрібні круглі дужки
const makeUser = name => ({ name: name, role: 'user' })
💡

Чому () навколо об'єкта? Без них JS думає що {} — це тіло функції, а не об'єкт. Дужки кажуть: "це вираз, не блок коду".

3 Що таке колбек

Колбек — це функція яку ти передаєш в іншу функцію. Та інша функція викличе її коли настане потрібний момент.

Аналогія: Ти замовив піцу і залишив свій номер телефону. Коли піца буде готова — тобі зателефонують. Твій номер — це колбек. Піцерія викличе тебе тоді коли потрібно. 
// setTimeout — чекає N мілісекунд, потім викликає колбек
setTimeout(() => {
  console.log('Минула 1 секунда')
}, 1000)

// addEventListener — чекає події, потім викликає колбек
button.addEventListener('click', () => {
  console.log('Кнопку натиснули')
})

// .map() — викликає колбек для кожного елементу
[1, 2, 3].map(num => num * 2)  // → [2, 4, 6]

// fetch — викликає колбек коли дані прийшли
fetch('/api/data')
  .then(res => res.json())       // колбек 1
  .then(data => console.log(data)) // колбек 2

4 Анонімна vs іменована функція

Іменована

function handleClick() {
  console.log('клік')
}

button.onClick = handleClick

Анонімна

button.onClick = () => {
  console.log('клік')
}

// пишемо прямо на місці

Коли що використовувати:

1

Іменована — якщо функцію використовуєш в кількох місцях. Пишеш один раз, викликаєш скрізь.

2

Анонімна — якщо функція потрібна тільки в одному місці. Зручно писати прямо всередині JSX чи методу.

5 Колбек з параметром event

Браузерні події автоматично передають об'єкт event в колбек. Він містить інформацію про подію.

// e або event — це обʼєкт з інфо про подію
input.onChange = (e) => {
  console.log(e.target.value)  // значення інпута
}

button.onClick = (e) => {
  console.log(e.target)  // елемент на якому клікнули
  e.preventDefault()     // скасувати стандартну поведінку
}

В React — те саме, але пишеш на компоненті:

// e автоматично передається React
<input onChange={(e) => setValue(e.target.value)} />

// Якщо event не потрібен — просто () =>
<button onClick={() => deleteAll()}>
⚠️

Важливо: якщо пишеш (item) => deleteTodo(item) в onClick — item буде event об'єктом, а не твоєю змінною з map. Завжди використовуй () => deleteTodo(item) щоб захопити потрібну змінну.

6 Замикання (closure)

Функція "запам'ятовує" змінні з того місця де була створена — навіть якщо виконується в іншому місці.

function makeCounter() {
  let count = 0  // ← ця змінна "захоплюється"

  return () => {
    count++
    console.log(count)
  }
}

const counter = makeCounter()
counter()  // → 1
counter()  // → 2
counter()  // → 3

В React ти вже використовував замикання — коли колбек в onClick використовує item з .map():

[...todos].map((item) => (
  <li key={item.id}>
    // item "захоплений" у цьому колбеку
    <button onClick={() => removeTodo(item.id)}>
      Видалити
    </button>
  </li>
))
