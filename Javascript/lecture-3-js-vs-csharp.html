JavaScript vs C#

Чим відрізняється підхід до функцій, класів і структури коду в JS порівняно з C#.

1 Головна різниця в підході

C# — об'єктно-орієнтована мова. Весь код живе в класах. JS — мультипарадигмова: можна писати з класами, без класів, функціонально.

C# Все в класах
// В C# без класу не можна
public class Calculator {
  public int Add(
    int a, int b
  ) {
    return a + b;
  }
}

var calc = new Calculator();
calc.Add(2, 3);
JS Просто функція
// В JS клас не потрібен
function add(a, b) {
  return a + b
}

add(2, 3)
// все, більше нічого
В C# функції — це методи класів. Вони не існують окремо. В JS функції — повноправні об'єкти. Можуть жити де завгодно.

2 Класи в JS — є, але не обов'язкові

JS має класи, але вони рідко використовуються в сучасному React. Для порівняння:

C# Клас
public class Hero {
  public string Name;
  public int Health;

  public Hero(
    string name,
    int health
  ) {
    Name = name;
    Health = health;
  }

  public void TakeDamage(
    int dmg
  ) {
    Health -= dmg;
  }
}
JS Клас
class Hero {
  constructor(name, health) {
    this.name = name
    this.health = health
  }

  takeDamage(dmg) {
    this.health -= dmg
  }
}

const hero = new Hero(
  'Axe', 100
)

В сучасному React — замість класів використовують об'єкти і функції:

// Замість класу — простий обʼєкт
const hero = {
  name: 'Axe',
  health: 100,
  takeDamage: (dmg) => hero.health -= dmg
}

// Або фабрична функція
function createHero(name, health) {
  return { name, health }
}

3 this — небезпечне місце

this в JS поводиться інакше ніж в C#. В C# this завжди вказує на поточний об'єкт. В JS — залежить від того як функцію викликали.

C# this завжди стабільний
public class Counter {
  private int count = 0;

  public void Increment() {
    // this завжди Counter
    this.count++;
  }
}
JS this залежить від контексту
const obj = {
  count: 0,
  inc: function() {
    this.count++  // ОК тут
  }
}

const fn = obj.inc
fn()  // this = undefined!
💡

В React з хуками проблеми з this практично немає — функціональні компоненти не використовують this. Це одна з причин чому хуки замінили класові компоненти.

4 Модулі замість namespace

В C# код організовують через namespace і класи. В JS — через модулі (файли) і export/import.

C# Namespace
namespace MyApp.Utils {
  public class MathHelper {
    public static int Add(
      int a, int b
    ) => a + b;
  }
}

// Використання
using MyApp.Utils;
MathHelper.Add(2, 3);
JS Модулі
// math.js
export function add(a, b) {
  return a + b
}

// App.js
import { add } from './math'

add(2, 3)
// просто функція, без класу

5 Типізація

C# — строго типізована мова. JS — динамічно типізована. TypeScript додає типи до JS.

C# Строга типізація
// Тип обовʼязковий
string name = "Axe";
int health = 100;

// Помилка компіляції якщо
// передати неправильний тип
string x = 42;  // ❌
JS Динамічна типізація
// Тип не вказується
let name = 'Axe'
let health = 100

// JS дозволяє змінити тип
name = 42  // ✅ дозволено!

// Помилка тільки в рантаймі
// TypeScript — JS з типами (як C#)
function add(a: number, b: number): number {
  return a + b
}

add('hello', 2)  // ❌ помилка TypeScript

7 Що взяти з C# досвіду

Знання C# — це перевага. Багато концепцій переносяться:

✅ Лямбди — майже ідентичний синтаксис

✅ async/await — та сама концепція

✅ LINQ методи — в JS це .map(), .filter(), .reduce()

✅ ООП розуміння — корисне для складних систем

✅ Типізація — TypeScript дасть знайомий досвід
